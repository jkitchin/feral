Cascade vs benign overlap completely on the symbolic ratio.
Hypothesis disproved: a pure symbolic auto-arm based on supernode
widths CANNOT distinguish cascade-prone from benign matrices.

Mechanism reason: the cascade emerges from delayed pivots
ACCUMULATING up the etree at runtime. The symbolic pass sees only
the pre-delay structure where every supernode is small (~400–1500
columns). The runtime root expansion to 21k cols (pinene iter 5
smoking gun) is invisible at symbolic time because delays are a
function of NUMERIC values, not pattern.

Alternative approaches that could work:
- (a) Runtime auto-arm: monitor n_delayed accumulation mid-factor
  and arm CB on remaining supernodes when n_delayed exceeds a
  threshold. Architectural change — params are immutable during
  factor today.
- (b) Warm auto-arm: record root supernode's n_delayed count from
  the previous factor() call; if it exceeded β*n, arm CB for the
  next factor on the same pattern. Requires persistent state and
  pattern-fingerprint guard. Lets the first factor pay the cascade
  cost but rescues iter 2+.
- (c) Etree-depth or aspect-ratio heuristic. Untested; cascade
  matrices may have wide-fanin trees that concentrate delays.

warm auto-arm as the only realistic single-shot fix without an
architectural change.

** Mechanism — why off by exactly 2 / 1:

BK forms 2×2 pivot blocks whose inertia is counted from det(D2):
det>0 → (2,0) or (0,2) by trace sign; det<0 → (1,1). When the
"true" 2×2 sub-block is nearly singular (det ≈ 0) one round-off-
scale perturbation can flip det's sign and mis-label a (+,+) block
as (+,−), shifting net inertia by (−1 positive, +1 negative). One
mislabel per affected 2×2 gives shift +/-1; two mislabels give +/-2.
The observed pattern (38402 at iter 1-3, 38401 at iter 4, 38400 at
iter 5+) matches "two mislabeled 2×2 blocks at iters 1-3, one at
iter 4, none from iter 5 onward as IPM moves into a more
well-conditioned region of the parameter space."

issue #17 (robot_1600), issue #18 (NARX_CFy), and the corpus's

principle unfixable without an external second opinion — but
matrices where MA57 and MUMPS agree on inertia, like rocket_12800
appears to be (MA57 makes consistent progress without restoration),
are the ones where feral *should* also agree.

implement an analogous perturbation.

factorization of the dumped matrix as written, and cannot be
corrected by changing how feral counts signs of D. It would
require either (a) implementing an MA57-style static pivot
perturbation in feral, or (b) confirming that the IPM driver
should be tolerating this 1-2-eigenvalue disagreement (e.g. by
bumping delta_w sooner on small inertia mismatches).

** Lesson for future investigations
"Off-by-N inertia with a residual at 1e-13" is not by itself
evidence of a sign-counting bug. By Sylvester's law a valid
factor's D must carry the inertia of A. To distinguish a real
sign-counting bug from a real eigenvalue near zero, recount
inertia independently from D (1x1 sign + 2x2 eigenvalues) and
inspect the smallest |eig| distribution. The §09:45 hypothesis
got published without that check; the §16:30 disproof would
have saved the implementation cycle.

* 10:58 :issue-38:static-pivot:capi:implementation:

Implemented MA57-style static-pivot perturbation knob for
issue #38. Adds NumericParams.static_pivot_threshold (Option<f64>,
default None = off) with semantics matching MA57 cntl(5):
relative threshold; the per-call absolute floor is
`t * ||A||_inf`, propagated into BunchKaufmanParams.static_pivot_floor.
For 1x1 pivots with |d| < floor, d is set to sign(d) * floor
(positive when d == 0). For 2x2 blocks, perturb_2x2_to_floor
computes both eigenvalues (sym2_eigenvalues, cancellation-free
discriminant) and if the smaller |lambda| < floor, shifts both
diagonals by tau = target - lambda_small where target =
sign(lambda_small) * floor (preserving sign of the small
eigenvalue). Both code paths set Factors.needs_refinement = true
so the solve path will run one mixed-precision refinement step.

Sign rule is *sign-preserving*, matching the conservative
MA57 cntl(5) default behavior (cf. Duff/Pralet 2007, eq. 14).
Sign-FLIPPING (which would actually shift feral's 38402 -> 38400
on rocket_12800 iters 1-3) was considered and rejected: it can
mis-label a genuinely negative pivot as positive on truly
indefinite KKTs (saddle-point check would silently fail).

C ABI exposure: FERAL_STATIC_PIVOT=<float> env var read at
feral_new() time; values <= 0 or non-finite are ignored.
