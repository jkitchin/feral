* 10:45 :phase2.3:experts:consultation:
Spawned =spral-expert= and =mumps-expert= in parallel to trace
the regression.

- =spral-expert= (SPRAL SSIDS): SSIDS's =ldlt_tpp_factor= handles
  unstable pivots by breaking out of the main elimination loop and
  leaving the remaining columns un-eliminated — the outer
  multifrontal driver then reassembles them into the parent. No
  zero-pivot-at-root fallback. The closest analog in feral would
  be to return Rejected with =nelim < ncol= at the root, which
  requires changes to inertia accounting and solve-side
  un-eliminated column handling.

- =mumps-expert= (MUMPS 5.8.2): MUMPS's =INFOG(28)= (number of null
  pivots detected) is only computed when =ICNTL(24) = 1= (null
  pivot detection). In the default configuration MUMPS hardwires
  =n0 = 0= regardless of what the matrix actually contains. So
  our oracle comparison saying "MUMPS detects 0 zero pivots on
  DEGENLPA and feral detects 1" is partly a measurement artifact
  — MUMPS isn't detecting zero, it's refusing to report any.
  MUMPS also has STATICMODE / CNTL(4)/CNTL(5) that perturbs tiny
  pivots to ±SEUIL and continues — OFF by default.

Conclusion: two root causes. (1) feral's ForceAccept-zero branch
converts small-nonzero pivots into zeros, losing sign. (2) MUMPS's
oracle is partly a lower bound, not ground truth, on zero-pivot
counts.

* 11:10 :phase2.3:fix:sign-preservation:
Applied the minimal fix at
=src/dense/factor.rs::try_reject_1x1_frontal=. Split the
=ForceAccept= branch into two cases:
- =|d| <= zero_tol=: truly zero. Count as zero_pivot, zero the L
  column, return Rejected. (Unchanged behavior.)
- =|d| > zero_tol=: small but nonzero. Accept with correct sign
  (=d > 0 → pos += 1=, else =neg += 1=), set
  =needs_refinement = true=, return Accepted so the caller runs
  the normal rank-1 update. Iterative refinement cleans up the
  small-pivot noise.

The 2×2 fallback routes through =try_reject_1x1_frontal=, so the
one fix covers both paths. =do_1x1_pivot= (dense-only) has no
=may_delay= parameter and is not reached from the frontal kernel
— not modified.
