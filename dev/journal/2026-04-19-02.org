* 14:00 :pounce:bench:cache-reuse:
Added src/bin/bench_solver_reuse.rs that demonstrates the β +
Solver win the per-matrix bench cannot show. For each of 4 KKT
families with N IPM iterates (capped at 64), runs two scenarios:
A) one persistent Solver, factor() each iterate; B) free-function
loop calling symbolic_factorize + factorize_multifrontal each
iteration. Reports symbolic_call_count, wall time, speedup.

Added warmup pass for both scenarios — first run was cold-cache
slow on the small ACOPP14 case (0.77x) and skewed the comparison.
With warmup:

| family   |   n |  nnz | iter | solver(us) | syms | freefn(us) | syms | speedup |
|----------+-----+------+------+------------+------+------------+------+---------|
| ACOPP14  | 106 |  492 |   21 |       1863 |    2 |       2569 |   21 |   1.38x |
| ACOPP30  | 209 |  765 |   64 |      11831 |    1 |      16553 |   64 |   1.40x |
| ACOPR30  | 400 | 1574 |   64 |      52500 |    2 |      58040 |   64 |   1.11x |
| AIRPORT  | 126 | 1928 |   64 |      11409 |    2 |      17516 |   64 |   1.54x |
| TOTAL    |     |      |      |      77603 |      |      94678 |      |   1.22x |

Observations:
- ACOPP14 / ACOPR30 / AIRPORT show 2 sym calls instead of 1: the
  pattern fingerprint (n, col_ptr_len, row_idx_len) catches an
  nnz change between iterates 0 and N. Acceptable — POUNCE will
  see structurally identical KKTs once warm-start kicks in.
- ACOPR30 shows only 1.11x speedup: numeric factor cost dominates
  (n=400, nnz=1574 vs. 209/765 for ACOPP30). Symbolic amortization
  matters less when each numeric factor is already large.
- Total 1.22x corpus-wide. Real value lands once we pair this with
  larger matrices in IPM mode where every saved symbolic is a
  meaningful fraction of total cost.

* 14:30 :bench:noise:beta:regression-investigation:
Investigated the 1.76 → 1.78 sparse-bucket p90 regression noted
in session 2026-04-19-02.

Setup analysis: bench.rs measures `sp_factor_us` as the wall time
across BOTH symbolic_factorize and factorize_multifrontal (line
1257-1285, single Instant). The β refactor moved compute_scaling
from symbolic → numeric, but both phases are inside that timer,
so total work should be unchanged.

Verified by 3 fresh release runs of `cargo run --release --bin
bench`:

| run | dense small | dense med | sparse small | sparse med |
|-----+-------------+-----------+--------------+------------|
|   1 |        1.52 |      1.91 |         1.89 |       1.90 |
|   2 |        1.48 |      1.83 |         1.83 |       1.83 |
|   3 |        1.47 |      1.86 |         1.86 |       1.86 |

Sparse p90 spread: 1.83 ↔ 1.89 = ±0.03 / 3%. The reported "1.76
→ 1.78 regression" is 1% — well inside the noise floor.

Conclusion: not a real regression. Closing the investigation.
Future bench reporting should average ≥3 runs (or report the
median + range) to avoid chasing single-run noise.
