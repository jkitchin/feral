* 20:10 :issue-9:phase-2.4.3:research-note:
Wrote =dev/research/block32-register-resident-kernel.md=. Key
findings:

1. SSIDS =block_ldlt= uses two-dimensional max-loc + =test_2x2= on
   the global uneliminated block, NOT column-relative BK. That is the
   APP-deferred-threshold kernel. The feral analogue must use feral's
   existing BK rule (gamma0 column max, alpha threshold, swap-1×1 /
   no-swap 2×2 / swap-2×2 / rejection) — direct port of SSIDS
   semantics would break the =parallel_corpus_parity= bit-parity
   contract.
2. Register-residency in Rust+pulp: one =WithSimd= dispatch wraps
   each kernel call, =const generic= BS=32 lets LLVM fully unroll the
   r-loop, no tail handling needed (BLOCK_SIZE % vlen == 0 for vlen
   in {2,4,8}). Per-call dispatch overhead ~30 cycles, amortized
   over the ~30-column trailing update → ~30× reduction in dispatch
   overhead per pivot vs the current per-axpy loop.
3. Expected end-to-end: 89 ns/nnz → ≤44 ns/nnz on CHAINWOO_0000
   (1.5× of SSIDS's 29 ns/nnz). Dense p90 vs MUMPS: 1.86 → ~1.5.

* 21:30 :issue-9:phase-2.4.3:tdd-step1-blas3:
Implemented =schur_panel_minus_nofma_strided_quad= in
=src/dense/schur_kernel.rs= (~340 LoC). Four-column variant of the
existing dual kernel: one pulp dispatch packs 4 destinations per
src-vector load, halving src bandwidth and doubling SIMD arithmetic
per load.

Design choice — **unroll=2 not 4** for the bulk SIMD body. 4 dst
accumulators × 4 chunks = 16 acc regs would spill on AVX2's 16-ymm
budget. Unroll=2 → 8 acc regs + 2 src + 1 alpha splat = ~11 live,
fits comfortably. AVX-512 under-uses its register file but the
dominant win is src-bandwidth halving, not register utilization.

Caps (3 rows that don't have entries in all four columns) processed
by scalar q-loops — max 192 scalar mul-subs per quad call, negligible
vs the bulk.

Bit-parity tests (2 new):
- =schur_panel_minus_nofma_strided_quad_is_bit_exact_vs_four_singles=
  sweeps 176 configs (n_elim × len0) against four sequential

- =schur_panel_minus_nofma_strided_quad_skips_zero_alphas_independently=
