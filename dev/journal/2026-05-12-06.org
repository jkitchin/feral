* 20:10 :issue-9:phase-2.4.3:research-note:
Wrote =dev/research/block32-register-resident-kernel.md=. Key
findings:

1. SSIDS =block_ldlt= uses two-dimensional max-loc + =test_2x2= on
   the global uneliminated block, NOT column-relative BK. That is the
   APP-deferred-threshold kernel. The feral analogue must use feral's
   existing BK rule (gamma0 column max, alpha threshold, swap-1×1 /
   no-swap 2×2 / swap-2×2 / rejection) — direct port of SSIDS
   semantics would break the =parallel_corpus_parity= bit-parity
   contract.
2. Register-residency in Rust+pulp: one =WithSimd= dispatch wraps
   each kernel call, =const generic= BS=32 lets LLVM fully unroll the
   r-loop, no tail handling needed (BLOCK_SIZE % vlen == 0 for vlen
   in {2,4,8}). Per-call dispatch overhead ~30 cycles, amortized
   over the ~30-column trailing update → ~30× reduction in dispatch
   overhead per pivot vs the current per-axpy loop.
3. Expected end-to-end: 89 ns/nnz → ≤44 ns/nnz on CHAINWOO_0000
   (1.5× of SSIDS's 29 ns/nnz). Dense p90 vs MUMPS: 1.86 → ~1.5.
