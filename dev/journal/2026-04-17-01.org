* 10:00 :contract:design:
Authored dev/plans/ordering-crate-contract.md. Three choices locked:
- i32 index width (matches Fortran/Ipopt convention, no 64-bit needs
  at this boundary).
- Free function per crate, no trait (four backends, diverging option
  types, zero-benefit indirection).
- No etree in the contract (three of four backends return a node
  separator tree, not an etree — forcing etree would shape-distort
  them).
