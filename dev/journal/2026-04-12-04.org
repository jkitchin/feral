* 10:15 :solve:refactor:
Refactored `solve_sparse` in `src/numeric/solve.rs`. Extracted the
existing body (AMD permute → forward sub → D-solve → backward sub
→ un-permute) into a private `solve_sparse_core` helper. Made
`solve_sparse` a thin wrapper that:
  1. Checks `factors.scaling_info != ScalingInfo::NotApplied` to
     decide whether scaling is needed (O(1) check, avoids the
     all-ones multiply on Identity/External paths).
  2. Pre-scales: `b_scaled[i] = rhs[i] * factors.scaling[i]`.
  3. Calls `solve_sparse_core(factors, b_scaled)` → `y`.
  4. Post-scales: `x[i] = y[i] * factors.scaling[i]`.
Note: same vector on both ends, not its inverse. Math from the
plan — `A·x = b` and `M = D·A·D` give `M·y = D·b` with `y = D^-1·x`,
so `x = D·y` recovers the user-space solution.

Did NOT touch `solve_sparse_refined`. Its correction solve loops
through `solve_sparse(factors, &r)`, so it inherits the pre/post-
scale wrapping transparently. Residual `r = rhs - matrix·x` still
uses the original unscaled `matrix` — exactly what refinement
needs. Read the code to confirm.
