* 18:05 :scaling:dense-fast-path:perf:
Shipped the small-n =compute_scaling= fast path.

Design: when =should_use_dense_fast_path= fires, the dense-fast-path
already has (or is about to produce) a =SymmetricMatrix= dense
buffer. Run Knight-Ruiz directly on that buffer instead of the
sparse =compute_infnorm= which strides through =row_idx[k]=
indirection on every entry.

Implementation:
- =compute_infnorm_dense(&SymmetricMatrix)= in src/scaling/infnorm.rs.
  Walks the lower-triangle in column-major order; bit-exact with
  sparse KR on any fast-path-gate matrix since CSC-stored entries
  and dense-stored entries cover the same nonzeros and the
  =if v > row_max[i]= guard makes the extra zero positions
  no-ops.
- =compute_scaling_dense_fast(matrix, sym, strategy)= in
  src/scaling/mod.rs. Routes Auto/InfNorm to the dense KR; honors
  Mc64Symmetric / Identity / External via the existing
  =compute_scaling= path.

  =to_dense_into= *before* computing the scaling, so the dense KR
  iterates the buffer that is about to be scaled in place.
