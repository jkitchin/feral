=cargo build --release= in =python/= failed with libpython link

* 01:00 :pyo3:contiguous-arrays:
=test_multi_rhs_solve= failed: "The given array is not contiguous"
when passing a numpy column slice =X[:, j]= into =solve=. =as_slice()=
requires contiguous storage. Fix: replaced every =as_slice()= call
on user-facing inputs with a copy via
=as_array().iter().copied().collect::<Vec<f64>>()= (helpers
=array1_to_vec= and =array1_i64_to_vec_usize=).

Conclusion: a Python binding has to assume non-contiguous inputs.
The copy cost is negligible compared to factor.

* 01:10 :ipm:zero-diagonals:
=test_kkt_solver_basic= failed: KktSolver tried to write δ_c into
the (2,2) zero block but =from_dense= had skipped those slots
(zeros aren't stored). Fix: changed test helpers to use
=from_triplet= with explicit =i == j= inclusion regardless of value,
plus =_kkt_with_explicit_zero_diag(K)= for ad-hoc cases. This is a
real API constraint: KktSolver requires that every diagonal entry
the caller intends to perturb be present in the pattern with a
(possibly zero) value. Documented in =KktSolver.__post_init__=.
