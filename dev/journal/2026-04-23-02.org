* 18:55 :rook-rescue:step-4:kernel:
Step 4 of the plan: replaced the =rook_rescue= stub in
=src/dense/rook.rs= with a full symmetric threshold-gated rook
alternating-scan kernel. Signature unchanged:
=fn rook_rescue(a, nrow, ncol, k, params) -> Option<RookPivot>=.

Algorithm (matches plan §3 and research note §3):

1. Init =r := k=, compute =(gamma_r, s) = sym_row_argmax(r)= over
   =[k, nrow)=.
2. Loop up to =MAX_ROOK_ITER = 8= (Ashcraft-Grimes-Lewis 1998
   safeguard):
   a. If =r < ncol= and =|A[r,r]| >= u * gamma_r=: accept 1×1 at r.
   b. Compute =(gamma_s, t) = sym_row_argmax(s)=.
   c. If =s < ncol= and =|A[s,s]| >= u * gamma_s=: accept 1×1 at s.
   d. If both r,s fully-summed and =gamma_s <= gamma_r= (i.e.,
      =|A[s,r]|= dominates both rows): try 2×2 at {r,s} subject to
      SSIDS det floor + Duff-Reid growth bound (both ported from
      =factor_frontal=:1409-1439).
   e. Advance =(r, gamma_r, s) := (s, gamma_s, t)=.
3. Return =None= on iteration exhaustion → caller falls through to
   delay/force-accept.

Gate choice: rook uses =params.pivot_threshold= (u, typically 0.01)
as the 1×1 accept threshold, NOT =alpha=. Research note §11 Q2
resolved this — splice-site semantics require the rescued pivot pass
the same u-threshold that rejected the original candidate.

Swap placement: =RookPivot::pair(k, r, s)= produces up to 2 symmetric
swaps that bring {r, s} to positions {k, k+1}, handling the special
case =r == k+1 && s == k= (single swap suffices) and tracking index
drift when the first swap relocates s.

Unit tests (5/5 pass in =cargo test --lib dense::rook=):
- =rook_accepts_1x1_at_row_2=: hand-traced Test 2 matrix; rook walks
  (0,0)→(0,1)→(2,1), then iter-2's gamma_s=100 <= gamma_r=100 at
  s=2 with =|A[2,2]|=500 >= u*100=1=, accepts 1×1 at row 2 via
  swap (0,2). Note the scan FIRST checks 1×1 at s BEFORE the 2×2
  termination test; this is the key design decision and is why Test
  2 produces a 1×1 rescue rather than the 2×2 that would fire if

- =rook_accepts_2x2_at_rows_1_2=: hand-traced Test 3 matrix; rook

- =rook_accepts_1x1_at_k_when_diagonally_dominant=: SPD with

Phase 2.4.3 Step 6 landed: Test 4 (Sylvester inertia preservation
under rook rescue) added to =tests/rook_rescue.rs=.

Property: for a random symmetric indefinite matrix, inertia is
invariant under pivot-sequence changes (Sylvester's law). Rook
rescue alters the pivot sequence on ill-conditioned matrices; the
test verifies it does not alter the inertia.

Design:
- Self-contained xorshift64 PRNG (identical to
  =tests/property_tests.rs=).
- =random_indefinite(n, rng)= constructs A = MMᵀ − mean(diag(A))·I
  then scales ~1/3 of diagonals by 1e-3 to drive their magnitude
  below =u * col_max=, forcing BK rejection. Off-diagonals preserved
  so rook can find rescue candidates along rows.
- Reference factor: =pivot_threshold = 0.0= (degrades threshold to
  =zero_tol=, never rejects) → BK-partial reference inertia.
- Rescue factor: =pivot_threshold = 0.1= (heavy; forces rescues).
- Sizes {6, 10, 16, 24, 32} × 5 trials = 25 matrices.
- Asserts: inertia equality; full elimination; no force-accept
  (=needs_refinement == false=); reconstruction residual
  =≤ 1e-10 × ||A||_F=; =total_rook_rescues > 0= across the sweep.

Initial attempt with unmodified =random_indefinite= did not trigger
any rook rescues at u=0.1 — the MMᵀ construction yields diagonals
that are comparable to row-max, so BK's swap + threshold test
accepts naturally. The 1e-3 diagonal-scaling step forces the
