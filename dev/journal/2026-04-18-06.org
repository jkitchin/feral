* 17:30 :kahip:k3:bugs:
Hit three K3 implementation/test bugs in sequence:

1. Grid_7x7 test oracle wrong — claimed stair-step cut ≥13, actual
   is exactly 12 (6 horizontal + 6 vertical crossings along r+c=6).

2. INF_CAP overflow. Used `i64::MAX/4` per pin; with many pins on
   super-source the cumulative `excess[source] -= cap` overflowed.
   Fixed: inf_cap = (sum_in_band_edge_weight / 2) + 1 (halved
   because edges array counts each undirected edge as two
   anti-parallel directed entries).

3. Fixed-node depth fallback. When a small part is fully contained
   inside the BFS ball, no vertex has dist==bnd_distance, so
   fixed_src_count or fixed_snk_count = 0 and K3 short-circuits to
   false. Fixed: pin_depth_p0 = min(max_dist_in_part_0,
   bnd_distance) and symmetrically for p1. Pins the deepest
   vertices actually present in each part.
