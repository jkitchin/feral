* 14:20 :kahip:k1:impl:tests-adjusted:
Implemented `crates/feral-kahip/src/data_reduction.rs` (~600 LOC
including tests). First compile passed, but 4/15 tests failed with
wrong oracles — my initial expected behavior didn't match what the
rules actually produce. Traces showed:
  - Star(n=10): Rule 1's cascade doesn't stop at the hub — once the
    hub loses all but one leaf, it becomes degree-1 too and cascades
    out. Any one vertex survives, not specifically the hub. Fix:
    test asserts bijection + op count, not which vertex survives.
  - Triangle + tail: triangles collapse via closed twins (all three
    vertices share closed_sig [0,1,2]), not "triangle survives".
  - Dumbbell with triangle endpoints: Rule 2 detects cycle-through-
    branch pattern (u==w when walking from a triangle vertex back
    to the hub) and skips; triangles then collapse via twins.
    Fix: test uses triangle + edge-through-middle configuration that
    exposes the chain to Rule 2 between distinct branch hubs.
  - K_{2,3}: Rule 2 fires BEFORE Rule 3 (twin) — degree-2 vertices
    on the size-3 side are path-compressed one at a time. After
    two compressions the residual is a triangle {0,1,4} which then
    collapses via closed twins. 2 path ops + 2 twin ops, not the
    "3 open twins" I guessed.

Also found a real bug: Rule 2's outer loop broke out entirely when
it hit u==w, instead of marking those vertices skipped and trying
other seeds. Fix: added a `skip[v]` array per-pass; when the
branch-endpoint walk yields u==w or u has degree 2 (pure cycle),
mark every vertex touched as skipped and continue to the next seed.
This is what unblocked the isolated-chain test — the left triangle's
cycle-through-hub-0 was causing the whole pass to bail before
reaching the actual 3-4 chain between distinct branch hubs 0 and 5.
