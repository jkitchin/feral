Plan mirrors SPRAL exactly for the non-singular path:
  1. Build full symmetric CSC with values (not reuse symmetric_pattern
     which doesn't carry values).
  2. Drop explicit zeros, take log|val|.
  3. Per column: cmax[j]=max, then entry ← cmax - entry.
  4. Call hungarian_match.
  5. rscaling[i]=u[i]; cscaling[j]=v[j]-cmax[j].
  6. s[i]=exp((rscaling[i]+cscaling[i])/2).
  7. Safety: clamp exp overflow, rewrite 0/NaN→1, unmatched→1.

For the partial-singular path, deviate from SPRAL: caller decided that
unmatched → s[i]=1.0 fallback is sufficient. Not implementing SPRAL's
reduced-matrix re-matching plus Duff-Pralet correction.

     before taking the log, so CSCs with structurally stored zeros
     (e.g. 2×2 pivot padding) are handled automatically. But a column
     whose ONLY entries are all zero will have cmax = NEG_INFINITY
     and fall back to identity for that index — this differs from
     "matched but with very small values" and is flagged in the code
     comments.
