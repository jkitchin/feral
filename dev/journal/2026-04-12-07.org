    IF ( abs(PIVOT).GE.UULOC*max(RMAX,AMAX)
  &     .AND. abs(PIVOT) .GT. max(SEUIL,tiny(RMAX)) ) THEN

And the 2×2 at lines 1590-1606:

    DETPIV = A(POSPV1)*A(POSPV2) - A(OFFDAG)**2
    ...
    IF ((abs(A(POSPV2))*RMAX+AMAX*TMAX)*UULOC.GT.ABSDETPIV ...)
      GO TO 460
    IF ((abs(A(POSPV1))*TMAX+AMAX*RMAX)*UULOC.GT.ABSDETPIV ...)
      GO TO 460

Both use UULOC = CNTL(1) = 0.01 by default (confirmed in
dfac_driver.F:472-480: CNTL(1)<0 → 0.01 for symmetric indefinite,
capped at 0.5). GOTO 460 is the delayed-pivot path. The 2×2 test
is a Duff-Reid growth bound: reject when u·(|a22|·RMAX + |a21|·TMAX)
> |det|. Feral has no analog — it tests only |det|·zero_tol_2x2.

* 18:20 :phase2-2-2:ssids-survey:
Read ref/spral/src/ssids/cpu/kernels/ldlt_tpp.cxx. The 1×1 test at
line 226 is:

    if( fabs(a[p*lda+p]) >= u*maxp ) { ... }

where maxp is find_rc_abs_max_exclude(p, nelim, ..., t) — max of
row/col p excluding the 2×2 partner. The 2×2 test in test_2x2 at
lines 89-119 is more interesting than MUMPS's because SSIDS has
THREE checks:
  1. maxpiv >= small (abs floor)
  2. |detpiv| >= max(small, |detpiv0|/2, |detpiv1|/2)
     — cancellation guard, the SSIDS-specific elaboration
  3. u·max(x1,x2) < 1 where x1 = |D^-1_11|·maxt + |D^-1_21|·maxp
     and x2 = |D^-1_21|·maxt + |D^-1_22|·maxp
     — Duff-Reid growth bound, same structure as MUMPS

Check (2) catches catastrophic cancellation in the 2×2 determinant
that (3) alone would let through. Neither MUMPS nor the BK77 paper
include it. Feral should replicate because it's cheap and exactly
addresses the 2×2 precision loss.

Default thresholds at ref/spral/src/ssids/datatypes.f90:260-262:
u = 0.01, small = 1e-20. Four orders tighter than f64::EPSILON.

Delayed pivoting: SSIDS returns nelim <= n from ldlt_tpp_factor; the
caller handles the remainder. Much simpler than MUMPS's INOPV/GOTO
460 mechanism, but still requires the caller to resize the parent.

* 18:40 :phase2-2-2:design-options:
Enumerated four options in the research note:
  A. Column-relative zero_tol + 2×2 Duff-Reid bound, no delayed
     pivoting (minimal)
  B. Option A + rewrite the BK77 1×1-vs-2×2 decision tree to match
     SSIDS literally (middle)
  C. Full delayed pivoting (textbook, weeks of work)
  D. Option A + MUMPS-style regularization on reject (hybrid)

Recommended **Option A**. Rationale:
  - It closes the ACOPP30 regression (rejected pivots get zeroed
    via ForceAccept, exactly as the Identity path's 1e-8 pivots
    do; the residual drops from 1e46 back to the Identity baseline
    ~1e16).
  - It leaves BK77 stability guarantees intact (new check is a
    superset of the existing check, not a replacement).
  - 1-2 days of work; fits the Phase 2.2.2 budget.
  - Matches SSIDS line 226 (1×1) and lines 89-119 (2×2) literally.
  - Does not diverge feral from the clean BK77 derivation the
    existing dense validation tests depend on.

Option C is the textbook-correct answer but belongs in Phase 2.3.
