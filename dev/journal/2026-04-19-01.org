* 09:00 :scaling:refactor:beta:
β refactor: moved scaling from symbolic to numeric phase.

Architectural goal: SymbolicFactorization must be cacheable across IPM
iterations of structurally identical KKTs. Before this change, MC64
scaling lived inside symbolic_factorize so every numeric refactor with
new values forced a re-symbolic pass. After: SymbolicFactorization is
purely structural; NumericParams { bk, scaling } is the per-numeric
bundle.

Removed from SymbolicFactorization: scaling, scaling_pivot_order,
scaling_info. Removed from SupernodeParams: scaling_strategy. Added
SparseFactors.scaling and .scaling_info on the numeric side.

New convenience constructor NumericParams::with_bk(bk) for callsites
that want default scaling. Default derived (caught by clippy).
