* 11:10 :amf:finalize-step-amf:pass2:
finalize_step_amf Pass-2 implements the three-accumulator
(deg, wf3, wf4) walk from the research note Section 6:
- element walk: dext = degree[e] - extent_in_me; first touch caches
  wf[e] = dext * (2*degree[e] - dext - 1); accumulates wf4 += wf[e]
  and deg += dext.
- variable walk: wf3 += nv[j], deg += nv[j].
- loose-degree branch (degree[i] < deg): zero wf3, wf4 to suppress
  the fill contribution from approximate-degree-tightened variables.
- otherwise degree[i] = deg; wf[i] = wf4 + 2*nvi*wf3.
Pass-1 lazy-zeroes wf[e] alongside the existing
we = degree[e] + wnvi line.

* 11:30 :amf:rmf-quantization:
The saturated/regular RMF branch at the re-insertion point uses f64
arithmetic to compute RMF = deg*(deg-1+2*degme) - wf[i] divided by
(nvi+1), then double-fallback quantizes:
  if rmf < dummy: qscore = rmf as i32 (rounded);
  elif rmf / n < dummy: qscore = (rmf / n) as i32;
  else: qscore = AMF_DUMMY_I32 (= i32::MAX - 1).
wf[i] = max(1, qscore). The dummy = i32::MAX - 1 cap matches MUMPS
ana_orderings.F:4230 (idummy = huge - 1). Saturated branch
(degree[i] + degme > nleft): take the min of the original and capped
RMF — protects against quantization stomping on degree-near-NLEFT
candidates.

* 12:15 :amf:supervar-merge:
Supervariable merge uses max: wf[i_u] = max(wf[i_u], wf[j]). The
research note Section 6 derives this from the worst-case fill
argument: when two variables have identical structure, eliminating
the merged supervariable produces fill at least equal to the worse
of the two component RMFs. AMD's merge is a no-op (degree carries
through nv accumulation); AMF needs the explicit step.

* 12:35 :amf:select-pivot-coarse-scan:
select_pivot_amf scans head[mindeg..]; at the first non-empty bucket
it inspects whether the bucket is in the coarse region (deg > n).
If yes: linear-walk the chain via next[] tracking the min-wf
candidate, doubly-linked unlink at the end. If no: take the head
directly (fine bucket — wf is exact, no scan needed). The amortized
cost stays O(N + nnz) because each variable only re-enters the
coarse-scan loop when its score crosses a coarse-bucket boundary.
