* 09:30 :investigation:perf:multifrontal:
Picked up from prior compaction. User confirmed "primarily a
performance issue, not an accuracy issue" framing. Diagnosis carried
over: numeric factor_nnz 19× over symbolic textbook L-fill on
PoissonControl K=158 (46.7M vs 2.4M). Worst-case at root supernode
1429 (first_col=7467, ncol=33): symbolic predicts nrow=33, numeric
reports nrow=3545, with 3512 rows < first_col. Only 33 rows are in
own_range — the remaining 3512 are upper-triangle leakage.

Root cause: ~build_row_indices~ at src/numeric/factorize.rs:2241-2308
iterates ~full_pattern.col_ptr[j]~ for j ∈ own_cols.
~full_pattern = matrix.symmetric_pattern()~ (csc.rs:186-239) is the
fully symmetrized A pattern, so each column carries both r > j
(legitimate trailing) and r < j (already-eliminated upper-tri)
entries. The latter pollute every frontal and propagate up through
child contrib blocks. Symbolic ~column_counts_gnp~ filters
~partner <= i~ correctly (column_counts.rs:135); numeric did not.

* 14:00 :investigation:factor-nnz:gap-analysis:
User asked next: investigate the remaining 1.6-2× gap between
numeric factor_nnz and Σ col_counts after the build_row_indices fix.
Specifically how much is delayed-pivot fill vs contribution-block
storage, and whether there's slack to recover.

Instrumented diag_poisson_kkt with a per-supernode breakdown:
true_L_nnz_in_own_cols (Σ col_counts in own range), sym_supernodal
padded (own_ncol*(own_ncol+1)/2 + (sym.nrow-own_ncol)*own_ncol), and
numeric_padded (factor_nnz formula). Two deltas: Δ_amalg = sym_padded
- true_L; Δ_passthrough = numeric_padded - sym_padded.

#+BEGIN_EXAMPLE
K=50:  Σcc=143667 sym_padded=131071 numeric=323643
       Δ_amalg=-12596 Δ_passthrough=+192572 (100% on n_delayed_in=0)
       0 delayed pivots, 1430/1430 supernodes have n_delayed_in=0
K=158: Σcc=2447001 sym_padded=2202355 numeric=4610269
       Δ_amalg=-244646 Δ_passthrough=+2407914
         of which +1741427 (72%) on n_delayed_in=0 nodes (13514/13868)
       367 delayed pivots, max=2 per supernode
#+END_EXAMPLE

Conclusions:

1. Delayed pivots are NOT the main source of the residual gap.
   K=50 has zero delayed pivots and exhibits the same 2.25× ratio.
   K=158's 367 delayed pivots only account for ~28% of the
   passthrough delta.

2. The dominant gap source is **pass-through row padding** in the
   dense trailing block. Children's contrib rows that aren't in any
   of the parent supernode's own-col L patterns get absorbed into
   the parent's frontal as zero rows × nelim cols, and then passed
   up to the grandparent. The numeric factor_nnz formula counts
   those zero entries as stored entries.

3. Δ_amalg is negative because Supernode.nrow at supernode.rs:358 is
   set from col_counts[first_col].max(ncol) — the L count for the
   FIRST column of the supernode, not the row union across all the
   supernode's columns. Cosmetic accounting artifact, not slack.

4. Worst-case on K=158: supernode 2537 with 24 children. sym says
   nrow=32 (no trailing) because col_counts[12635]=32; numeric finds
   nrow=148 because the 24 children push 116 trailing rows through.
   116×33 = 3828 zero entries stored in this single supernode's
   trailing block.

5. Slack analysis: this gap is normal for dense supernodal
   multifrontal storage and matches MUMPS/SSIDS reported ratios.
   Three mechanisms could narrow it but none are free wins —
   amalgamation tuning (nemin), compressed L storage (slower
   solves), or better ordering (ND vs AMD; per Issue #3 to be
   re-evaluated post-fix).
