* 15:00 :issue-42:inertia:rankdef:f01:diagnosis:
Picked up issue #42 — feral reports inertia (4,5,1), zero=1, on the
synthetic stress matrix rankdef_10_3, agreeing with no canonical oracle
(MUMPS ICNTL(24)=1 zero=3, SSIDS zero=0, MA57 zero=0). The stress gate
(report.py) requires feral.zero ∈ {mumps.zero, ssids.zero} = {3, 0};
zero=1 is allowlisted under #42.

Ran probe_f01 (already in-tree) to dump feral's actual per-pivot |d|
on rankdef_10_3. Result:

  inertia=(4,5,1)  null_pivot_tol = sqrt(n)·EPS·||A_scaled|| = 4.337e-15

  k=7  |d|=5.551e-15  d=+5.551e-15  SIGN-ACCEPT (above tol) -> +
  k=8  |d|=8.075e-16  d=-8.075e-16  F-01 BAND (sign-fallback) -> -
  k=9  |d|=0.000e0    d=+0.000e0    STRICT_ZERO (case a)     -> zero

So zero=1 is NOT a threshold-band artifact. It is the count of
*bit-exactly-zero* pivots: exactly one of the three constructed-null
pivots (the trailing 1×1 Schur complement, k=9) reduces to a true 0.0
under feral's elimination order. The strict-zero rule |d| ≤ EPS
correctly catches it. The other two near-null pivots reduce to tiny
nonzero floats (5.55e-15, 8.08e-16) and are counted by sign per the
#39 sign-fallback design — which deliberately left the strict-zero
case (a) path unchanged (f01-rankdef-underreporting.md:309 "Strict
zeros (|d| ≤ EPS) are still zeros").

Note k=7 at 5.551e-15 is ABOVE the current null_pivot_tol 4.337e-15,
so even reverting the #39 sign-fallback (count whole band as zero)
would yield zero=2, not 3. MUMPS-IC24's zero=3 requires a larger
relative threshold than feral's sqrt(n)·EPS·||A||.

Why the oracles differ:
- MUMPS-IC24=3: explicit null-pivot detection, relative threshold
  catches all three (5.5e-15, 8.1e-16, 0.0).
- SSIDS=0, MA57=0: default sign-counting; their elimination order
  produces no bit-exact-zero pivot, so all three near-nulls are
  tiny-nonzero and counted by sign.
- feral=1: hybrid — detects the one bit-exact zero, counts the two
  near-zeros by sign.

Conclusion: this is a genuine design decision, not a localizable bug.
feral is consistent within itself (#39 chose the sign-counting lane;
case (a) only ever catches true 0.0). The hybrid count is the
unavoidable consequence of "count by sign EXCEPT bit-exact zeros."
Three resolution paths exist (zero=0 via counting exact zeros by sign;
zero=2/3 via relative rank detection, contradicts #39; or change the
gate / exclude the matrix). Surfacing to the user for the decision.
