#+TITLE: 2026-04-27 Session 04 — W-3b: move contrib into ContribBlock

* Goal
W-3b from =dev/plans/dense-kernel-speedup.md=: eliminate the
=ff.contrib.clone()= the multifrontal driver was performing on every
supernode. For CHAINWOO_0000's 1984-row root supernode, the
contribution block is cdim² = (1984-32)² × 8 bytes ≈ 30 MB; cloning
that on every factorization is pure malloc traffic.

The headline gain remains modest because the CHAINWOO root supernode's

allocator stress under repeated IPM-style factorizations) than for
single-run wall time.

* Decision
Take the =mem::take= approach rather than introducing a borrowed
output buffer in the kernel API. Reasons:
1. Zero LoC change in the kernel — the in-place output is already
   produced as a freshly-allocated =Vec=; we just transfer ownership
   instead of cloning.
2. Tests that read =ff.contrib[...]= directly via =factor_frontal*=
   (delayed_pivoting.rs, etc.) keep working unchanged.
3. Pooling the =contrib= buffer through =FactorWorkspace= would be a
   larger refactor (the kernel would need to take an
   =&mut Vec<f64>=). Defer to a future session if measurements show
   it matters.

Started =dev/research/amf-clean-room.md= with mumps-expert in

Key findings landed in the note:

- HAMF4 is a 400-600 LoC delta on top of AMD, NOT a 1500-LoC
  clean-room. Same quotient-graph machinery, same hash detection,
  same Scan-1/Scan-2 skeleton; six inner-loop sites change.
- New data: one =WF= array of length N, doubling as fill score (for
  variables) and cached element fill area (for elements).
- New buckets: =NBBUCK = 2*N=, =PAS = max(N/8, 1)=, coarse
  quantization above =NORIG= because RMF can be O(N^3).
- AMF metric (Amestoy 1999 thesis):
    RMF = DEG*(DEG-1+2*DEGME) - WF(i)
    WF(i) = sum_e dext(e)*(2*deg(e)-dext(e)-1) + 2*NVI*sum_j nv(j)
- Recommended architecture: extract =feral-ordering-core= with a
  =Metric= trait; =feral-amd= and =feral-amf= become thin
  specializations sharing one quotient-graph code path.
- Test oracle: nnz_L from symbolic factorization, target
  feral-amf nnz_L within 10% of MUMPS HAMF4 (109,782 on ORBIT2).
  Byte-deterministic perm match infeasible because tie-breaking
  depends on linked-list insertion order.
- Estimated scope: 3-5 sessions (module factoring + AMF impl +
  oracle plumbing + corpus validation).
