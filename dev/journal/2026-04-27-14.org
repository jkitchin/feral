* 14:00 :b1:dual-kernel:design:
Picking up Phase B-1 of dev/plans/dense-kernel-blas3.md after
Phase A (W-2 2x2 inline) shipped in session-13. Plan calls for a
register-blocked MR×NR DSYRK micro-kernel; pursuing a narrower
NR=2 dual-column variant first as a stepping stone.

Design: process two adjacent trailing columns (j, j+1) in lockstep,
sharing src loads. Key observation: dst0[1..] (column j rows
[j+1, nrow)) and dst1[..] (column j+1 rows [j+1, nrow)) reference
the **same** src memory at column offset
`(k+q)*nrow + j + 1`. So a single src load per chunk per q feeds
both accumulator stacks.

Cap: dst0[0] (column-j diagonal entry) processed by scalar
prologue. Bit-exact with rank-1 reference per element.

* 14:30 :b1:kernel-implementation:
Wrote `schur_panel_minus_nofma_strided_dual` in
src/dense/schur_kernel.rs (~210 LoC). Structure mirrors the
existing `schur_panel_minus_nofma_strided`:
- 4-way unrolled main body holding 8 SIMD registers (4 dst0 + 4 dst1)
- tail full-lane chunks
- masked partial-load tail

Inner per-q does shared src load, then independent
`mul -> sub` chains per accumulator. Zero-alpha skip honored
independently per column to preserve bit-exactness with two
sequential single-kernel calls.
