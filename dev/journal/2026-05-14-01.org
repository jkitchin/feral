* 09:55 :f2.3:design:
Decision: kappa_1 is a property of (A, factor), not of the
refinement iterate. Computing it once per refinement run (3-5
extra solves) is correct; computing it per step would multiply
that cost by the iteration count for zero new information.

Per-step diagnostic is the Skeel forward-error bound
=kappa_1_est * relative_residual= — this *does* change per step
because relative_residual drops as refinement progresses.

API shape: new public function =solve_sparse_refined_with_diagnostics=
mirroring =solve_sparse_refined= exactly in control flow, returning
=(Vec<f64>, RefinementDiagnostics)=. Refactored the existing
function into a private =solve_sparse_refined_core(..., with_diagnostics: bool)=
so both surfaces share one implementation. F2.3 mandate is "no
behavior change" — same iterate must come out either way.
