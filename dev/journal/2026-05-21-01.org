278ms. Classic delayed-pivot accumulation into the root front.

So the two mechanisms have OPPOSITE profile signatures:
- Mechanism A (robot/marine): loop-dominated; cost piles into a
  few large/root supernodes via delayed pivots.
- Mechanism B (rocket): prologue-dominated; the numeric loop is
  already fast. NOT covered by phase-2.11 small-front
  amalgamation (that targets loop-side tiny-front overhead).

Static read of the prologue (factorize.rs:1668-1782) shows only
O(n)/O(nnz) code — symmetric_pattern, permute_csc_values,
from_triplets are all linear. 3.4s for nnz=333k is ~100-1000x
too slow for the apparent work. The cost is empirical and needs
sub-phase instrumentation to localize — that is plan step B1.
