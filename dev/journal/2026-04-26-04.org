#+TITLE: 2026-04-26 Session 04 — Streaming bench redesign

* Goal
Session 03 left the expanded-corpus sparse pass blocked by the
bench's upfront-load architecture: =FERAL_KKT_ROOTS=all= loads CSC
for all 167,614 matrices simultaneously, then iterates dense + sparse
loops. The 10,120 n>1000 matrices push the sparse-loop entry working
set over the 64 GB laptop ceiling and SIGKILL the bench during the
sparse iteration.

Streaming redesign goal: load metadata only at startup, then in each
loop iteration parse the .mtx on demand, do the work, drop. Peak
memory bounded by single-largest-matrix rather than corpus
cumulative.

* 21:30 :plan:
Two-pass streaming approach (lower-risk than single-pass):
- Shrink =KktEntry= to metadata only (=name=, =mtx_path=, =sidecar=,
  =mumps_timing=, =ssids_timing=). Drop =matrix= and =csc= fields.
- =load_kkt_dir= becomes meta-only: walks subdirs, reads .json sidecar
  + oracle JSONs, skips matrices missing sidecar or with NaN/Inf RHS.
  Does NOT parse .mtx.
- Dense loop body opens .mtx, validates NaN/Inf entries + dim, calls
  =to_dense=, runs factor + solve, drops.
- Sparse loop body opens .mtx (again — penalty: 167k extra parses),
  converts to CSC, runs symbolic + multifrontal + solve, drops.
- =print_perf_comparison= and =print_phase28_partition= switch from
  =&[KktEntry]= to the same =&[KktEntry]= post-refactor (signature
  unchanged, struct just lost matrix/csc fields).

Cost: each .mtx parsed twice on the dense-eligible portion. For 154k
small matrices that's ~seconds extra. For the 10k large matrices it
matters more (~minutes), but those don't go through dense path
(n > 1000) so they're parsed once for sparse. Net: ~few minutes
extra runtime, vs unbounded crash today.

Single-pass would save the duplicate parse but requires merging the
output structure of the two loops. Two-pass preserves output
ordering and contains the diff. Choosing two-pass.

* 21:50 :impl:
Refactor done. Diff:
- =KktEntry= shrunk to =name + mtx_path + sidecar + mumps_timing +
  ssids_timing= (5 fields, was 7). Dropped =matrix= and =csc=.
- =load_kkt_dir= now metadata-only: reads .json sidecar + oracle JSONs,
  filters on missing/bad sidecar + NaN/Inf RHS. Does NOT parse .mtx
  or run NaN/Inf-entry / dim / csc validations.
- Dense loop body parses .mtx, runs entry-finiteness / dim /
  =to_dense=, factors, solves, drops at end of scope.
- Sparse loop body parses .mtx, runs entry-finiteness / dim /
  =to_csc=, drops mtx, runs symbolic + multifrontal + solve, drops
  csc at end of scope.
- New parse-skip counters (=n_dense_parse_skipped=, =sp_parse_skipped=)
  surfaced in summaries when nonzero. Pass-rate denominators subtract
  parse-skipped to preserve old semantics.

Build clean. =cargo test --release= 333 pass.
