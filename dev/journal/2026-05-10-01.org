* 14:30 :issue-5:bk-pivoting:reproducer:
Reproduced the issue #5 wandering inertia symptom on MSS1_0000 in a
unit test inside =src/numeric/factorize.rs::tests=. The reproducer
sweeps δ_w ∈ {0, 1e-4, 1e-2, 1, 1e2, 1e4, 1e6, 1e8, 1e10, 1e12}
adding to the (1,1)-block diagonals (first 90 entries). Configuration
matches ripopt's =feral_direct.rs= exactly: =Identity= scaling,
=ForceAccept= zero-pivot, =zero_tol=1e-10=, =zero_tol_2x2=1e-20=,
=pivot_threshold=0.0=.

Key finding: the corpus =MSS1_0000.mtx= has Ipopt's static
regularization δ_c = -1e-8 baked into the (2,2) block. With it in
place, every δ_w in the sweep produces a clean =(90, 73, 0)=. Strip
it (matching the reporter's =dc=0= trace), and the wandering appears:

#+begin_example
δ_w =      0e0  →  inertia(+ 93, - 48, 0: 22)
δ_w =     1e-4  →  inertia(+ 91, - 46, 0: 26)
δ_w =     1e-2  →  inertia(+ 94, - 49, 0: 20)
δ_w =      1e0  →  inertia(+ 90, - 45, 0: 28)
δ_w =      1e2  →  inertia(+ 92, - 47, 0: 24)
δ_w =      1e4  →  inertia(+ 90, - 45, 0: 28)
δ_w =      1e6  →  inertia(+ 90, - 45, 0: 28)
δ_w =      1e8  →  inertia(+ 90, - 45, 0: 28)
δ_w =     1e10  →  inertia(+ 90, - 45, 0: 28)
δ_w =     1e12  →  inertia(+ 90, -  1, 0: 72)
#+end_example

Mathematical truth: with K = [[(1+δ_w)·I, J^T], [J, 0]] and J having
structural rank ≈ 45, the *correct* inertia at every finite δ_w is
=(135, 45, 28)=, independent of δ_w. The Schur complement is
S = -(1+δ_w)^-1·J·J^T which has rank 45 throughout. So:

  - δ_w = 1, 1e4, 1e6, 1e8, 1e10 produce (90, 45, 28) — the n+ count
    is wrong (should be 135) but n- and zero are right.
  - δ_w = 0, 1e-4, 1e-2, 1e2 wander: n+ flips between 90 and 94.
  - δ_w = 1e12 collapses: Schur entries underflow to ~0 and BK reads
    pivots as zero (72 zeros). Real failure mode.

The ripopt =PDPerturbationHandler= relies on monotone non-decreasing
n+ as δ_w increases, expecting eventually n+ → n_x = 90. With the
wandering it sees n+ = 91 → 94 → 90 etc. and ramps to 1e12 hoping to
pin it. This breaks the IPM Newton step.

Two distinct phenomena are conflated in the trace:
1. =δ_w ∈ [0, 1e10]=: BK 1×1/2×2 boundary instability — pivot
   selection flips at the α-test (|akk| ≷ 0.6404·γ_0) as δ_w sweeps
   through the magnitude of off-diagonal J^T entries.
2. =δ_w = 1e12=: Schur underflow. (1+δ_w)^-1·J·J^T entries fall below
   =zero_tol=, BK declares zero pivots.

Both are addressed by Option B (norm-relative pivot floor) in the
research note: a SEUIL-style floor catches the wandering pivots with
an absolute threshold, and routes the underflowed pivots to a
sign-preserving zero count rather than letting them silently flip
between 1×1 sign assignments.

Test currently asserts =any_decrease= on the positive count (locks in
the symptom). Once Option B lands, that flips to assert monotone
non-decreasing.

* 15:30 :issue-5:option-b:invalidated:
Two diagnostic sweeps invalidate the research note's Option B
(norm-relative pivot floor / SEUIL analog) as a fix for the wandering
symptom.

** zero_tol sweep (n+ trace by zero_tol)
#+begin_example
zero_tol = 1e-10:  [93, 91, 94, 90, 92, 90, 90, 90, 90, 90]
zero_tol =  1e-8:  [93, 91, 94, 90, 92, 90, 90, 90, 90, 90]
zero_tol =  1e-6:  [93, 91, 94, 90, 92, 90, 90, 90, 90, 90]
zero_tol =  1e-4:  [93, 91, 94, 90, 92, 93, 90, 90, 90, 90]
zero_tol =  1e-2:  [93, 91, 94, 90, 92, 90, 90, 90, 90, 90]
#+end_example
Identical n+ trace across 8 orders of magnitude of =zero_tol=. The
wandering pivots are O(1), not below the floor. Option B's
absolute-floor lever does not touch the wandering at small δ_w; it
only pulls the large-δ_w underflow tail into the zero bucket
(visible as n- collapsing earlier and zero count rising at large δ_w).

** pivot_threshold sweep (n+ trace by u)
#+begin_example
u = 0:     [93, 91, 94, 90, 92, 90, 90, 90, 90, 90]   range 4
u = 1e-10: [96, 98, 97, 91, 101, 98, 94, 101, 96, 90] range 10
u = 1e-8:  [96, 98, 97, 91, 101, 98, 94, 98, 90, 90]  range 10
u = 1e-2:  [96, 98, 97, 91, 104, 98, 94, 96, 90, 90]  range 13
u = 0.5:   [93, 95, 96, 92, 93, 98, 95, 94, 93, 90]   range 6
#+end_example

Raising =pivot_threshold= (issue #2's lever) makes the wandering
*worse*, not better. With u=0 the spread is {90..94} (4); with u in
the recommended {1e-8 .. 0.1} range the spread blows out to {91..101}
(10). At u=0.5 (Bunch-Kaufman's classical value, ripopt's =pivtol_max=
cap) it settles at {92..98} (6). None achieve monotonicity.

** Conclusion
The wandering is not a magnitude-threshold problem. It's BK pivot
ordering ambiguity on a structurally rank-deficient J: each
floating-point perturbation of the input picks a different valid
factorization with a different (pos, neg, zero) categorization. All
the sweeps reach the *correct* inertia (90, 45, 28) at most δ_w
values, but stochastically jitter ±5 around it depending on which
pivots were committed in which order.

The =δ_c = -1e-8= baked into the corpus matrix breaks this
degeneracy: with that 1e-8 in place, every δ_w produces clean
(90, 73, 0). That's Ipopt's static regularization doing exactly
what it's designed for.

** Implications for the fix
Option B (Norm-relative pivot floor) is *not* the right intervention
for this symptom. The actual fix is one of:

1. =Caller-side= — ripopt should add δ_c > 0 from iteration 0 to
   break the rank-deficiency before passing the KKT to feral. This
   matches Ipopt's standard practice and is the canonical IPM
   prescription. *No feral-side change required.*
2. =Force-1×1 mode= — add a flag to feral that disables 2×2 pivot
   selection entirely, so degenerate cases get split into 1×1's with
   well-defined sign assignments. Less stable in growth but more
   stable in inertia. Useful as an "inertia-correction-friendly"
   mode an IPM can opt into during δ_w escalation.
3. =Eigenvalue-aware 2×2 split= — when a 2×2 candidate has a
   smaller eigenvalue below SEUIL, split it into a 1×1 (with the
   larger eigenvalue) plus a 1×1 zero. Requires explicit
   eigendecomposition of the 2×2 block. Intermediate complexity.

Recommend pause + ask the user which direction to pursue. The
reproducer test + diagnostics are committed; pivoting work waits on
that decision.

* 14:45 :issue-5:test-design:regression:
The test is a *negative-control regression test* — it passes today
because the bug is present, and will fail when the bug is fixed. The
docstring on =issue_5_mss1_iter0_inertia_wanders_under_delta_w_sweep=
explicitly tells the next agent to flip the assertion when Option B
lands. This is preferable to =#[ignore]=-ing a positive-control test
that requires the fix because (a) the fix can be developed
incrementally without breaking CI, and (b) reverse-incident reviewers
get an immediate red flag on regression.

Skipped silently (=return=, not =panic!=) if the corpus matrix is
absent — keeps the test usable in the published crate where
=data/= is excluded.
