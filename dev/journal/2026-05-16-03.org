* 15:00 :bk:theory:thresholds:
Read `src/dense/factor.rs` thresholds. Two thresholds:

  - `zero_tol` = `f64::EPSILON ≈ 2.22e-16` (absolute floor for "true zero").
  - `pivot_threshold * col_max` (column-relative). NumericParams::default()
    sets `bk.pivot_threshold = 1e-8`, so the relative threshold here is
    ~1e-7 (col_max ≈ O(10) for these matrices).

Neither threshold ever fires because the random orthonormal Q spreads
the small λ=10^-p across all rows. Each pre-factor `min |A_ii|` is
already ≥ 4e-5 (mostly 1e-3 or larger). BK's emerging Schur complement
diagonals never get small. Standard story — Higham 2002 Ch. 11 gives
the bound |d_k| ≥ (1-α²)σ_min(A_22), and dispersing one mode across
n directions leaves σ_min(A_22) ~ O(1) until the last block.

* 15:20 :decision:no-default-change:
Decision: do NOT change the default thresholds and do NOT propose
trailing-norm criterion. Reasons documented in the research note
section "Proposed criterion (rejected)":

  1. Trailing-norm probe would flag p∈{6,7,8,9} as rank-deficient when
     MUMPS/SSIDS/Pardiso all report rank-100. Would violate the
     CLAUDE.md inertia gate.
  2. MA27/HSL precedent: report inertia of D, not inertia inferred
     from a separate condition probe. Users have
     `estimate_condition_1norm` for rank certification.
  3. Residual gate already passes uniformly at machine precision —
     factorization is informationally complete.
