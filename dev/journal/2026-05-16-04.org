* 13:40 :research:saddle-inertia:
  The issue text quotes the saddle inertia formula as
  =(n, n - dim ker(A), 2 dim ker(A))= but that double-counts:
  =n + (n - k) + 2k = 2n + k= which does not equal the matrix size
  =n + m = 2n - k=. Rederived from the standard saddle-inertia
  result (Gould 1985, Forsgren-Murray 1993): for K = [H A^T; A 0]
  with H ≻ 0 and A an m×n constraint of rank m-r, the inertia is
  =(n, m-r, r)=. Total =n + m - r + r = n + m= ✓.
  Used the corrected formula in the generator; oracle matches
  =np.linalg.eigvalsh= exactly on both saddle_rankdef sizes
  (50_10_3 → (50,37,3), 100_20_5 → (100,75,5)).

* 14:05 :tried-and-rejected:mc64-resistant:
  First mc64_resistant attempt was =A = I + α u u^T= with u = 𝟙/√n
  and α = -2, which I claimed would give cond ~ 1e8 after MC64
  scaling. Wrong: the eigenvalues of that rank-1 update are 1
  (mult n-1) and 1+α = -1, so cond(A) = 1 and there is nothing for
  MC64 to fail at. Verified with =np.linalg.cond= on the assembled
  matrix: 1.48 before scaling, unchanged after. Switched to a Q D Q^T
  construction with one eigenvalue at 1e-8 in a dense random basis;
  this is the canonical "MC64 sees an O(1) diagonal and produces
  s ≈ 1, so scaling does not equilibrate" setup. Verified
  cond = 2.0e8 before *and* after a symmetric row-max scaling proxy.
