* 15:30 :phase-2.3:step-5:build_row_indices:
Rewrote build_row_indices in src/numeric/factorize.rs to produce the
three-region layout [own_cols | delayed_cols | trailing_sorted]:
- Walks each child's ContribBlock.row_indices[..n_delayed] to harvest
  delayed-column global indices in child-iteration order.
- Builds an is_fully_summed lookup (indexed by matrix column) so the
  trailing-row BTreeSet can filter out any delayed column that also
  shows up as a pattern row of a parent column. Necessary because
  the full symmetric pattern includes transposed entries, so a
  delayed column can appear as a row of an unrelated parent column.
- Trailing set is the union of (own cols' pattern rows) and
  (each child's non-delayed contrib rows), minus fs_set.

When every child has n_delayed == 0 this code is point-wise
equivalent to the old implementation: delayed_cols is empty, fs_set
equals own_cols, and the BTreeSet filter reduces to the old
"not in [first_col..first_col+ncol)" predicate.

* 15:45 :phase-2.3:step-5:contrib-store:perm:
Replaced the `row_indices[ncol..].to_vec()` contrib store with a
perm-unpermuted form:

    for cj in 0..cdim {
        contrib_rows.push(row_indices[ff.perm[nelim + cj]]);
    }

Reasoning: BK pivot search only swaps within [0..expanded_ncol), so
ff.perm[i] = i for all i >= expanded_ncol. That means for the
trailing-row slice of the contrib block (positions n_delayed..cdim),
the formula reduces to row_indices[nelim + cj] = row_indices[ncol + cj]
for cj >= n_delayed — the same as the old straight-slice form. For
the delayed slice (positions 0..n_delayed), ff.perm[nelim + cj] is
the post-swap original position of that column in [0..ncol), and
row_indices[..ncol] is the ordering we fed the kernel, so the
lookup recovers the global index correctly.

Verified by running factorize_multifrontal_delayed_pivot_succeeds_at_parent
(test 1.5) which builds a 4×4 arrow matrix whose column 0 fails the
column-relative test at threshold 0.01 and must be delayed to the
root. Sparse inertia matches dense LDLᵀ inertia exactly.

* 16:00 :phase-2.3:step-5:may_delay-flag-flip:is_root:
Computed is_root[snode_idx] once at the top of factorize_multifrontal
by walking symbolic.supernodes[*].children — a node is a root iff no
other supernode claims it as a child. Handles disconnected matrices
(multiple roots) uniformly. Flipped the factor_frontal call to
`may_delay = !is_root[snode_idx]`. Non-roots get the delay path;
roots stay on ForceAccept.
