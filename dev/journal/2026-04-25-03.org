* 16:00 :phase-2.10:conclusion:
Conclusion: the tiny-IPM tail is overwhelmingly an *amalgamation*
problem, not a kernel problem. Closing this gap means either
(a) an aggressive small-front merge pass at symbolic-factorization
time (mirroring MUMPS's KEEP(197)=1 + NEMIN=5), or (b) a fast-path
kernel that avoids per-front allocator round-trips when nrow ≤ N
(mirroring MUMPS's NASS<24 single-shot path). Phase 2.11 should
prototype (a) first because it is closer to MUMPS's actual behavior
and the histogram suggests the wins are large.

Acceptance invariants for Phase 2.10 satisfied:
- count_sum == n_supernodes (no warnings)
- bucket sum_us == loop_us (no warnings)
- prologue + loop + epilogue ≤ total_us across all 5 matrices

MUMPS JOB=4 means "analyze + factor" combined. The reported
`factor_us` includes both phases. *Methodology is apples-to-
apples.*

Confirmed feral side too: =src/bin/bench.rs= line 1445-1476
explicitly times symbolic + numeric together as one block,
with a comment stating this is to match MUMPS/SSIDS semantics:

#+begin_example
// Symbolic factorization. Timed together with the numeric phase so
// the reported `factor_us` is apples-to-apples with the single
// `factor_us` field MUMPS and SSIDS emit for their equivalent work.
let tf = Instant::now();
... symbolic_factorize ...
... factorize_multifrontal ...
let sp_factor_us = tf.elapsed().as_micros();
#+end_example

*Conclusion.* The KIRBY2_0007 9.5× ratio is real. MUMPS does
analyze+factor in 122µs for n=458; feral does the same work in
~1160µs (904µs symbolic + ~235µs numeric). The bench is fair;
the gap is primarily feral-amd taking 770µs on n=458 where MUMPS
gets the entire combined operation done in 122µs.

Methodology asymmetry note: feral uses min-of-N resample for fast
matrices (=should_resample=); MUMPS oracle is single-shot. This
*favors feral*, so it cannot explain a feral-disadvantage ratio.

Phase 2.13b plan steps 1+2+3 are now complete. The remaining

- (a) symbolic caching across same-pattern factorizations (IPM-

- (b) shrink feral-amd per-call constant. Targets ordering only.
