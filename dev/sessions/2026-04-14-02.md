# Session 2026-04-14-02

## Goal

Phase 2.4 performance follow-up: land a SIMD Schur-update micro-kernel
in the dense LDLᵀ factorization to reach the Phase 2.8 exit criterion
(dense factor p90 ≤ 2.0 and sparse factor p90 ≤ 3.0 vs MUMPS oracle)
without any correctness regressions.

## Accomplished

### Phase 2.4.2 — FMA unroll4 wire-in (reverted)

Wired a pulp-dispatched NEON `axpy_minus_unroll4` / `axpy2_minus_unroll4`
kernel (using `mul_add_f64s`) into `do_1x1_update` / `do_2x2_update` in
`src/dense/factor.rs` behind a runtime `USE_SIMD_SCHUR_KERNEL` AtomicBool
flag. Microbench showed ~1.8× speedup on the Schur inner loop.

KKT bench result was worse than baseline on correctness: **4 sparse
inertia regressions** (ACOPP14_0001, ACOPP30_0004, FBRAIN3LS_0848,
FBRAIN3LS_0851) vs the pre-SIMD scalar baseline, with none going the
other way.

Triage: built `examples/triage_sparse_inertia_diff.rs` that factors
each of the 154588 matrices twice back-to-back (flag false, flag true)
and classifies into `{both_match, scalar_only, simd_only, neither}`.
Confirmed `both_match (153005) + scalar_only (4) = 153009` exactly
reproduces the scalar baseline total. The 4 regressions are pure
FMA-vs-scalar 1-ULP rounding deltas that push a 2×2 determinant or a
1×1 pivot through the `zero_tol` rank-classification boundary:

- `mul_add_f64s(−α, s, d)` = `round(−α·s + d)` (one rounding)
- scalar `d[i] -= α·s[i]`  = `round(d − round(α·s))` (two roundings)

Per the hard rule *"correctness before performance, always"*, reverted
the wire-in: `src/dense/factor.rs` back to byte-identical HEAD, triage
binary + AtomicBool flag removed. Kept the kernel module, microbench,
and pulp dependency in-tree for the Phase 2.4.3 retry.

### Phase 2.4.3 — non-FMA unroll4 (shipped)

Added `axpy_minus_unroll4_nofma` and `axpy2_minus_unroll4_nofma` in
`src/dense/schur_kernel.rs`. Same 4-way independent accumulator
structure, but per-lane ops use `simd.mul_f64s` + `simd.sub_f64s`
(and `simd.add_f64s` for the 2×2 variant's m0+m1 fold) — no FMA,
so per-lane rounding is bit-identical to the scalar reference.

Proved bit-exactness by unit tests (`assert_eq!`, not ULP4) across
`LENGTH_SWEEP = [0, 1, 2, 3, 4, 5, 7, 8, 9, 15, 16, 17, 31, 32, 33,
63, 64, 65, 127, 128, 129, 255, 256, 257, 511, 512, 513, 1023, 1024]`
and two seeds. All 13 schur_kernel tests pass.

Wired into `factor.rs` `do_1x1_update`/`do_2x2_update` with no runtime
A/B flag — because nofma is bit-exact with scalar, there is nothing to
A/B compare. Dispatch uses the direct const
`NEON: pulp::aarch64::Neon = unsafe { Neon::new_unchecked() }` pattern
to bypass the `#[target_feature]` trampoline.

## Decisions Made

See `dev/decisions.md` Phase 2.4.3 entry:
**"Schur SIMD kernel must use separate mul + sub, not FMA."**
Any future SIMD kernel that participates in a code path whose numerics
flow into pivot-classification thresholds must match scalar rounding
per-lane. FMA is permissible only in pure-throughput paths where the
result is not tested against a tolerance that was calibrated on scalar
rounding.

## Abandoned Approaches

See `dev/tried-and-rejected.md` Phase 2.4.2 entry:
**"Phase 2.4.2 unroll4 FMA Schur-kernel wired into do_1x1/do_2x2
(reverted)."** FMA's single-rounding semantics caused 1-ULP deltas at
the `zero_tol` pivot boundary, regressing 4 sparse inertias.
