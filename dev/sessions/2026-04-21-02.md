- **Step 2 â€” Compression module.** `src/symbolic/ldlt_compress.rs`
  defines the `SuperMap` struct and the three operations
  `build_supermap(perm)`, `compress_pattern(pat, map)`,
  `expand_permutation(super_perm, map)`. Emission order is
  deterministic (discovery order through `build_supermap`'s cycle walk).
- **Step 3 â€” Cycle extraction.** `build_supermap` walks the MC64
  matching permutation and decomposes 1-cycles â†’ singletons, 2-cycles
  â†’ one pair, k-cycles (k â‰¥ 3) â†’ `âŒŠk/2âŒ‹` pairs + optional singleton.
  Unmatched columns (`perm[j] == usize::MAX`) become singletons.
  Matches MUMPS `DMUMPS_SYM_MWM`.
- **Step 4 â€” Pattern compression.** `compress_pattern` contracts each
  stored edge `(i, j)` onto `(icmp[i], icmp[j])`, drops self-loops,
  and dedups parallel edges using a `Vec<u32>` tag-reset marker
  accumulator (same pattern as AMD) to avoid per-column full resets.
- **Step 5 â€” Permutation expansion.** `expand_permutation` emits pair
  originals adjacent in `(pair.0, pair.1)` order, so downstream BK
  sees each tentative 2Ã—2 pair in consecutive columns.

  path: if `map.ncmp() == n` (no compression leverage â€” pure 1-cycle

## Decisions Made

- **Wiring pattern: field on `SupernodeParams`.** Chose option (A)
  from the plan â€” one orthogonal `preprocess` flag crosses with any
  `OrderingMethod`. Keeps `OrderingMethod` focused on the actual
  ordering algorithm and avoids an NÃ—2 enum explosion.
- **Kept as opt-in; did not flip default.** Evidence is strong on
  the high-leverage bucket but the random sample is dominated by
  tiny matrices (< 100 variables) where symbolic overhead matters
  more than factor savings. A proper bucketed bench with residual
  parity tracking is a follow-up before flipping.
- **Fast-path on `ncmp == n`.** When the MC64 matching is purely
  on-diagonal (no compression leverage), fall through to the
  uncompressed path instead of running the graph contraction. This
  keeps the 53% `compRat = 1.0` corpus bucket cost-free when the
  preprocess flag is on.
