## Decisions Made
- Two-pass streaming over single-pass: dense and sparse loops each
  re-parse the .mtx. Cost: extra parses on the dense-eligible 157k
  matrices (~seconds). Benefit: preserves output ordering, contains
  the diff to load_kkt_dir + the two loop bodies.
- `FERAL_SPARSE_MAX` default `usize::MAX` (no cap) preserves
  historical kkt-corpus behavior. Setting it is opt-in for the
  expanded-corpus pass.

## Abandoned Approaches
- Single-pass streaming (rejected): would save the duplicate parse
  but requires merging the output structures of dense and sparse
  loops. Two-pass keeps the diff small.
