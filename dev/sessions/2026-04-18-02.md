### 4. Adversarial A1-A10 tests (commit `32bbb3b`)

Added 9 new tests in `feral-metis/src/fm_refine.rs` (A1 was

| case | structure | invariants enforced |
|------|-----------|---------------------|
| A2 | P_20 alternating | I1, FM improves from 19 |
| A3 | C_12 alternating | I1, FM improves from 12, floor ≥ 2 |
| A4 | 4×4 checkerboard | I1, FM improves from 24, floor ≥ 4 |
| A5 | 6×6 SplitMix-seeded | I1, I2 (determinism) |
| A6 | K_{4,4} imbalanced 1/7 | I1, I2 (softened from I4 — see below) |

| A8 | P_10 all A (empty side) | I1 (no panic) |
| A9 | n=1 | short-circuit |
| A10 | empty edge set | no moves, determinism |

**Surprise on A6.** Initial A6 asserted I4 (balance after FM from
a 1-7 imbalanced start with `max_imbalance=0.5`, `side_max=7 >
max_side=6`). Test failed because `refine_bisection` leaves the
labels at 1-7. Traced: `best_cut` initialises to the pass input
cut at `best_prefix=0`; when the FM trajectory first reaches a
balanced state, `cur_cut < best_cut` must hold for rollback to
adopt it. Classic FM assumes balanced initialisation — this is
design, not a bookkeeping bug, and the research note §5 already
flagged it. A6 was softened to I1+I2 with an explanatory comment.
The expected post-fix value "FM does not rebalance an imbalanced
start" is now baked into the regression suite. A genuine
follow-up candidate (rebalance pre-pass) is recorded below.

Evidence: 51 tests in `feral-metis` (was 42, +9); all green.

## Decisions Made

- **`OrderingMethod` is an enum, not a trait.** Three options on
  the table (enum, trait, generic-over-method). Enum chosen for
  the three reasons: only three implementations exist today,
  `symbolic_factorize_with_method(&matrix, &params, method)` is
  the ergonomic call site (no type parameter on the caller), and
  `match method` keeps the cross-crate conversion adapters in
  one file rather than requiring each ordering crate to know
  about `FeralError`. Trait-dispatch would cost more boilerplate
  and give nothing the enum doesn't. Recorded in
  `dev/decisions.md`.
- **Large-matrix corpus sourced from SuiteSparse, not synthesized
  or sampled.** Four matrices pinned via a manifest + fetch
  script; data gitignored. Rationale: real published matrices
  are reproducible, citable, and don't require synthesizing a
  structure we'd then debate the realism of. The alternative
  (mine IPM dumps from `data/matrices/` or `ripopt` runs) was
  rejected because those are already covered — poorly — by the
  parity corpus, and adding more of the same shape does not
  close the n > 10k gap. Recorded in `dev/decisions.md`.
- **In-tree `src/ordering/amd.rs` is not retired in this
  session.** Retirement requires reconciling the FeralError ↔
  OrderingError boundary and the owned-usize ↔ borrowed-i32
  pattern conversions at every call site. Filed as a deferred
  decision; no behaviour change required. Recorded in
  `dev/decisions.md`.
