### Expert consultation

Spawned `spral-expert` and `mumps-expert` in parallel to trace the
DEGENLPA_0065 regression. Root causes identified:

1. **feral's ForceAccept-zero branch converts small-negative pivots
   into zeros, losing sign information.** In
   `src/dense/factor.rs::try_reject_1x1_frontal`, when a pivot failed
   the column-relative test at a root (`may_delay = false`), the
   fallback unconditionally incremented `zero` and zeroed the L
   column, regardless of whether `|d|` was above or below the
   absolute floor `zero_tol ≈ eps`. On DEGENLPA_0065 this converted
   a small-but-clearly-negative pivot into a zero, producing
   `(20, 14, 1)` instead of `(20, 15, 0)`.
2. **MUMPS's INFOG(28) is only computed when ICNTL(24) = 1.** In
   the default configuration the reference's `n0` is hard-coded to
   0, so the MUMPS oracle never reports zero pivots even when the
   matrix is numerically near-singular. Our oracle comparison is
   therefore partly a measurement artifact — MUMPS isn't really
   "detecting" zero inertia more often, it's refusing to report
   any.
3. **SSIDS's approach:** break at the first unstable pivot and
   leave the rejected column un-eliminated (the outer multifrontal
   driver reassembles it to the parent). This preserves sign
   correctness by never force-accepting a bad pivot at a root.

### Sign-preservation fix (commit `a630977`)

Applied the minimal fix: split the `ForceAccept` branch in
`try_reject_1x1_frontal` into two cases.

- `|d| <= zero_tol` — truly zero, rank-deficient. Count as zero,
  zero the L column, return `Rejected`.
- `|d| > zero_tol` — small but nonzero. Accept with the pivot's
  sign (`d > 0 → pos += 1`, else `neg += 1`), set
  `needs_refinement = true`, return `Accepted` so the caller runs
  the normal rank-1 update. Iterative refinement cleans up the
  small-pivot numerical noise.

The 2×2 fallback routes through the same function, so the one fix
covers both paths. Separately-used `do_1x1_pivot` (dense-only,
no `may_delay` parameter) was audited and left alone — it's not
reached from the frontal kernel.

## Decisions Made

Appended to `dev/decisions.md`:

- **Pivot-threshold split between dense and sparse callers**
  (2026-04-13). The dense path uses `pivot_threshold = 0.0` because
  its BK kernel does not implement delayed pivoting and runs under
  Knight-Ruiz equilibration which handles scaling concerns at
  preprocess time. The sparse multifrontal path uses
  `pivot_threshold = 0.01` (SSIDS / MUMPS default `u`) so the
  column-relative test fires and the delayed-pivoting machinery
  from Steps 5+6 has something to reject. `BunchKaufmanParams::default()`
  stays at `0.0` to preserve the invariant that `factor()`
  library clients see unchanged behavior across Phase 2.3. Sparse
  callers explicitly override the default.

- **Root-fallback pivot sign preservation** (2026-04-13).
  `try_reject_1x1_frontal` splits its `ForceAccept` branch so that
  small-but-nonzero pivots at a root supernode are accepted with
  their correct sign (not converted to zero), and
  `needs_refinement` is flagged so iterative refinement can clean
  up the residual. Only `|d| <= zero_tol` (essentially `eps`) is
  counted as a zero pivot. Rationale: converting small-nonzero
  pivots to zero loses inertia information and produces residuals
  that iterative refinement cannot recover (the pivot was driven
  to 0 instead of being preserved with its noisy-but-nonzero
  value). Matches SSIDS's sign-tracking and is a strictly smaller
  change than SSIDS's break-at-root approach.

## Addendum: Refinement-Termination Fix (post-checkpoint)

After this checkpoint was written, the 6 remaining residual-margin
failures were triaged via `examples/triage_residual_margin.rs`.
The shared cause was `solve_sparse_refined` exiting early on a
`|dx|/|x| < eps*sqrt(n)` threshold before the trajectory settled
into the machine-precision basin. Under ForceAccept factorizations
the trajectory is non-monotone — corrections produce small `dx`
without reducing `r`, so `dx` is a false convergence signal.

Fix (commit `ed07ee3`):
- `max_steps`: 3 → 10 (matches MUMPS ICNTL(10) default).
- Termination: `||r|| < eps*sqrt(n)*||b||` (residual-based), with
  divergence guard intact.
- `||b|| = 0` handled with absolute threshold. Critical: do NOT
  clamp `||b||` to a floor of 1.0 — CERI651C has `||b|| = 3.238e-5`
  and the clamp makes the target absolute, fires at step 0, and
  defeats the entire refinement loop.

Applied identically to `src/dense/solve.rs::solve_refined`.

Parity: 22/28 → 27/28. Un-ignored AVION2_0510, CERI651C_0746,
CERI651ELS_1482, HAHN1_0004, MEYER3NE_0253. Only SSI_2597
remains ignored — trajectory stuck at 1.80e-13 regardless of
iteration count, a factorization-level limitation on a
pathological matrix (n=3, denormal values, 10^331 dynamic
