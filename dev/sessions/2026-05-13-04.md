### Diagnosis of remaining feral losses (and triage)

- **Initial hypothesis (symbolic-cache miss): falsified.** Env-gated
  `FERAL_C_API_TRACE` diagnostic in `src/capi.rs` confirmed the
  `PatternFingerprint` cache hits perfectly: `sym_calls=1` across
  50+ factor calls on LukVlI5. The bottleneck is per-call numeric
  factorisation, not symbolic re-analysis. Diagnostic reverted
  before committing.
- **Confirmed hypothesis (wide / near-square supernodes).** Env-gated
  `FERAL_PROFILE_FIRST` profiler attached to the first IPM iterate of
  `MBndryCntrl_3D` vs `MBndryCntrl_3D_27` (both at 24 mesh nodes per
  dim, n=31104). Result table is in the issue #14 body. Headline:
  - 7-pt stencil (`_3D`): tall-skinny supernodes (typical ncol 16–28),
    sequential factor 1474ms, top supernode 12ms.
  - 27-pt stencil (`_3D_27`): wide / near-square supernodes (several
    at ncol 169–433, root at 1928×1928), sequential factor 2456ms,
    root supernode 152ms alone, snode 3593 (2829×433) is 196ms.
  - Back-of-envelope on snode 3593: ~250 MFlop / 196ms ≈ 1.3 GFLOPS
    sustained, vs Apple Accelerate DGEMM at 50–100 GFLOPS on these
    shapes. MA57's advantage is delegating Schur updates to a tuned
    BLAS — a path explicitly closed off by feral's "no BLAS / LAPACK"
    constraint.
- **Filed issue #14** ("Wide / near-square supernode throughput")

## Decisions Made

- **C ABI lives in `feral::capi`, not a separate workspace member.**
  Earlier-today decision to put the ABI in a sibling `feral-capi`
  crate was superseded during implementation: the ABI is small
  (7 functions, ~250 lines) and tied 1:1 to types already public
  in the core crate. Collapsing to `pub mod capi` + `crate-type =
  ["staticlib", "rlib"]` keeps the FFI surface in one auditable
  file without an extra workspace member. Documented in
  `dev/decisions.md` 2026-05-13.
- **`feral-ipopt-shim/` lives in-tree during bring-up.** Single-PR
  C-ABI-coupled changes win during churn; split criteria recorded
  for when the ABI reaches semver 1.0 or we need multiple shim
  variants. `dev/decisions.md` 2026-05-13.
- **MBndryCntrl_3D_27 dense-kernel gap is parked as long-term polish
  (issue #14).** Closing it requires a near-peak GEMM-equivalent
  inner kernel — a months-of-work investment, not a few-days fix —
  and the share of the problem distribution it serves is smaller
  than the tall-skinny regime where feral now wins. Not a hard
  decision (no code change) but worth recording: the per-iteration
  gap on `MBndryCntrl_3D_27` is an acknowledged limitation, not a
  regression.
