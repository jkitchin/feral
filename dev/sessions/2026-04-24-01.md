# Session 2026-04-24-01 — Phase 2.9 SmallLeafSubtree (implemented, gated Off, no speedup)

## Goal
Close the 10× per-front-overhead gap on long-tail IPM matrices
(ACOPR30, CRESC100, HAIFAM) diagnosed in the prior session, by
batching true-leaf supernodes into subtree-units. Phases A–F of
`dev/plans/phase-2.9-small-leaf-subtree.md`.

## Accomplished

**Symbolic (Step A):**
- New module `src/symbolic/small_leaf.rs` with `SmallLeafParams`,
  `SmallLeafGroup`, and `find_small_leaf_groups`. Qualifies
  `children.is_empty() && ncol <= 8 && nrow <= 16` and packs into
  postorder-consecutive groups under a 4096-f64 arena budget.
- Populated unconditionally on `SymbolicFactorization` as
  `small_leaf_groups: Vec<SmallLeafGroup>` and
  `snode_group: Vec<Option<usize>>`.
- 8 unit tests, all passing. Integration with existing 143 symbolic
  tests: all pass.

**Numeric (Step B):**
- `factor_one_small_leaf` — leaf specialisation of
  `factor_one_supernode`. Takes precomputed row indices, skips
  `build_row_indices` and the (empty for leaves) extend-add loop.
- Dispatch loop in `factorize_multifrontal_supernodal_with_workspace`
  dispatches grouped supernodes to the specialised path and falls
  through to the scalar path for everything else.

**Gate (Step C):**
- `SmallLeafBatch::Off | On` enum on `NumericParams::small_leaf`,
  default `Off`. Zero behavioural change for existing callers.

**Parity (Step D):**
- `tests/small_leaf_parity.rs`: 7 tests, all pass. Bit-exact parity
  between Off and On across block-diagonal SPD fixtures,
  ACOPR30_0067, CRESC100_0000, HAIFAM_0082, VESUVIO_0000. Plus a
  sanity check that grouping actually kicks in on archetype
  matrices (10 grouped per matrix or more).

**Bench (Step E):**
- `src/bin/diag_small_leaf.rs`: release-build min-of-5 per config.
  Results below.

**Gate flip (Step F):**
- **NOT performed.** Speedup bar was 3×; measured geomean ≈ 1.00×.
  Default stays `Off`.

**Test suite:** 151 lib tests pass; 50+ integration tests all pass;
clippy clean under `cargo clippy -- -D warnings` (exact pre-commit
config).

## Decisions Made

- **Keep the Phase 2.9 implementation in the source tree, gated
  Off.** The specialised numeric path and the symbolic grouping
  pass cost nothing when disabled and are prerequisites for
  Phase 2.9.2 (true stack-arena, MUMPS-style). Ripping them out
  now and re-adding later is wasteful.
- **Do not flip `SmallLeafBatch::default()` to On.** Step F of the
  plan required ≥ 3× on ACOPR30; measured 1.00×.

## Abandoned Approaches

- **Naive "precompute row indices + skip empty children loop" as
  the SmallLeafSubtree speedup lever.** Correctness-preserving
  but performance-neutral — the per-front overhead is not in
  `build_row_indices`. Full writeup appended to
  `dev/tried-and-rejected.md` (section dated 2026-04-24).

## Next Session Should

1. **Diagnose where the per-front overhead actually lives.** The
   1220 µs / 341-supernode factorize on ACOPR30_0067 is ~3.6 µs
   per front. Profile candidates:
   - `frontal_buf.resize(n*n, 0.0)` memset cost at small n.
   - `factor_frontal_blocked` kernel setup overhead on ncol ≤ 8.
   - BK pivot-search + `FrontalFactors` allocation (perm/perm_inv
     vecs per front).
2. **Phase 2.9.2 sketch:** true shared-arena allocation. Refactor
   `factor_frontal_blocked` to accept `&mut [f64]` backing slice
   instead of owning a `SymmetricMatrix`. Members of a leaf group
   then write into one contiguous slice that gets memset once per
   group.
3. **Run the full 154k-matrix bench with the gate off** and confirm
   no regression vs the pre-phase-2.9 baseline — a belt-and-braces
   check that the dispatcher fall-through is truly zero-cost.
4. Consider whether the symbolic grouping pass cost is justified
   given the feature doesn't ship enabled. Its cost is O(n) and
   cached on `SymbolicFactorization`, reusable across IPM
   iterations, so the cost-per-flop is tiny; keep it.
