**Honest reading of the numbers:** the aspirational geomean
target of 1.15× assumed the BK 1×1-vs-2×2 search dominates
per-pivot time. The measured 1.025–1.05× across runs shows
that on the small-to-medium synthetic shapes used here, the
shared rank-1 trailing-update kernel (`do_1x1_update`) is
what actually dominates, and skipping the pivot search saves
only a modest constant per column. Run-to-run noise also
flips the sign on `medium-dense` and `large-banded` (~5%
noise band on shapes < 1 ms).

**The primary value proposition is robustness, not speed.**
The Vanderbei (1995) Theorem 2.1 diagonal-D guarantee is
*pivot-search-free*: it does not depend on a BK 1×1-vs-2×2
search finding a stable enough pivot. For matrices near the
BK pivot threshold — interior-point KKT systems as the
barrier parameter shrinks, ill-conditioned saddle systems
from constrained QP — the SQD path can complete cleanly
where BK is forced into 2×2 pivots, rook rescues, or
delayed-pivot cascades. The modest speedup is a bonus; the
real win is a predictable code path that either succeeds
under a stated contract or aborts loudly. Use cases:

1. IPM inner loops where every factor is structurally
   identical and the user has independent confidence in the
   SQD contract — predictable factor time matters more than
   a single-digit-percent speedup.
2. Saddle-point regularizers (Friedlander-Orban 2012,
   Pougkakiotis-Gondzio 2020) where the SQD form is
   guaranteed by construction and BK pivot rescues would be
   noise on a path that should never need them.
3. Anywhere "factor will succeed iff contract holds" is a
   more useful surface than BK's "factor will succeed,
   possibly with delayed pivots or perturbations, returning
   refinement-needed flag."

## Decisions Made

- **SQD ship targets demoted from gates to aspirations.** The
  `bench_sqd` binary prints PASS/MISS but exits zero on
  functional success, so CI can run it without flapping on
  per-machine noise. Targets retained as documented goals; a
  future commit may revisit with corpus-scale SQD inputs.
  Recorded in `dev/decisions.md` 2026-05-16 entry.
- **`Solver::with_sqd_mode(true)` retains opt-in semantics.**
  Default `sqd_mode = false` keeps the BK path as the
  universal default. Callers (IPM inner loops, POUNCE QP
  pre-conditioning) must opt in and accept the contract-trip
  surface.
