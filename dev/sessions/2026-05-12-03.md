## Accomplished

This session shipped **three iterations** of the same investigation
thread, each falsifying a prior hypothesis with telemetry:

1. **Iteration 2 — within-scope localization.** Added
   `task_wall_ns` (whole-closure bracket) and `ws_lock_wait_ns`
   (per-worker workspace mutex) to `AtomicLockStats`. Derived
   `rayon_idle = scope·T − task_wall_agg` and
   `ctrl_flow_agg = task_wall_agg − body_agg − all_locks`. Findings:
   - cont-201's 14.2 ms cached-mode gap is **78% rayon idle**
     (12.3 ms/worker, etree dependency wait), 10% locks, 10%
     control flow. **Etree-topology bound, not engineering loss.**
   - c-big at T=4 is essentially sequential: 74% rayon-idle
     capacity, 1.04× speedup over `body_agg`. The parallel
     multifrontal driver does not help c-big.
   - Per-worker workspace mutex wait is rounding error
     (≤2.5 ms across all matrices, ≤0.02 ms/worker on c-big).
   - **Closes the cont-201 assembly-tree-parallelism
     investigation**. Within-supernode parallelism (panel-BK /
     threaded dense kernels) is the only remaining axis;
     blocked behind Phase 2.4.3 SIMD bit-parity work.
   - Commit: `c51ea91`.

2. **Iteration 3 — scaling-cache verification.** Added the
   `solver_scaling_phase_split` probe test that loads each corpus
   matrix and times `pick_scaling_strategy`,
   `compute_scaling_with_cache(cache=None)`, and the reorder
   gather separately. Findings:
   - c-big picks `Mc64Symmetric`; no-cache path takes 2.3
     seconds (full Hungarian matching), cached path in
     production takes 2.4 ms. **1000× cache hit, cache works.**
   - cont-201 and bcsstk38 pick `InfNorm`. ~4 ms is fundamental
     Knight-Ruiz iteration on matrix VALUES. **Not cacheable
     across IPM iterations.** The 3.95 ms scaling slice on
     cached cont-201 is fundamental per-factor work, not a
     missed cache.
   - **Closes scaling probe** (#2 from prior session). No
     engineering action needed.
   - Commit: (this session, after writing this checkpoint).

3. **Decisions logged** to `dev/decisions.md`:
   - 2026-05-12 — Reject lock-free contribution-block store
   - 2026-05-12 (b) — Defer within-supernode parallelism; close
     cont-201 assembly-tree investigation as etree-bound
