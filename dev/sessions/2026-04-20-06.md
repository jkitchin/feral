# Session 2026-04-20-06

## Goal

Phase 2.4.1b Step 5 — enable the blocked panel path for
`may_delay == true` (SSIDS-style delayed pivoting used by non-root
supernodes). Before this session `factor_frontal_blocked` short-circuited
to `factor_frontal` whenever `may_delay == true`, so the blocked kernel
was unreachable on the supernodal driver path. Step 5 wires the delayed-
pivot `break on first failure` contract through the panel so the blocked
kernel can later replace `factor_frontal` inside
`src/sparse/multifrontal/`.

## Accomplished

### Tests-first: 3 new parity tests under `may_delay == true`

`tests/blocked_ldlt.rs`:
- `test_may_delay_spd_parity_size_sweep` — SPD at n ∈ {32, 64, 100, 128,
  200, 300}, `may_delay=true`. SPD never rejects, so the panel runs to
  completion; just validates the wiring doesn't break the happy path.
- `test_may_delay_ncol_lt_nrow_parity` — `nrow=200, ncol=128`,
  `may_delay=true`. The supernode case where the panel crosses the
  block boundary.
- `test_may_delay_rejection_parity` — n=128 with a dominant off-diagonal
  at column 32 under `pivot_threshold=0.01`. Forces a Delayed outcome;
  scalar breaks at k=32 with `nelim=32, n_delayed=96`. Blocked must
  produce byte-identical output.

Pre-implementation (with the `may_delay` short-circuit still in place)
all three passed trivially via delegation. After removing the short-
circuit they exercise the real blocked `Delayed` path.

### Implementation

`src/dense/factor.rs`:
- **`PanelStatus::Delayed`** — new third variant. Produced when a 1×1
  pivot fails the column-relative threshold under `may_delay=true`.
- **`factor_frontal_blocked`** (line ~822): gate changed from
  `may_delay || bs < 2 || ncol <= bs` to `bs < 2 || ncol <= bs`. Outer
  loop now matches on 3-way `PanelStatus`:
  - `Full` → continue.
  - `ScalarFallback` → one `scalar_pivot_step`, possibly break.
  - `Delayed` → break (caller of `factor_frontal_blocked` picks up the
    carried-forward columns from the `contrib` block).
  - `j_start = k + n_elim + 1` for **both** `ScalarFallback` and
    `Delayed` (same double-update avoidance as Step 4b).
- **`lblt_panel_frontal`** signature: takes new `may_delay: bool`
  parameter. The hard-coded `/* may_delay = */ false` in the
  `try_reject_1x1_frontal` call is replaced with the passed-in value.
  The `PivotOutcome::Delayed` match arm, previously a "this is
  unreachable" error, now returns `(c, PanelStatus::Delayed)`.

The key correctness claim: when `PivotOutcome::Delayed` fires,
`try_reject_1x1_frontal` has NOT mutated state. The peek-ahead applied
pivots 0..c-1 to column `k+c`, matching scalar's state at the break
point (scalar applies each pivot's rank-1 eagerly to all trailing
columns, so at break time column `k+c` has pivots 0..c-1 applied via
`do_1x1_update`). `apply_blocked_schur` with `j_start = k+c+1` applies
the same updates to columns `[k+c+1, nrow)` — bit-exact with scalar by
the same ascending-pivot replay order argument as Step 4b.

## Decisions Made

- **`PanelStatus` becomes 3-variant** (`Full`, `ScalarFallback`,
  `Delayed`). `Delayed` is only produced when the caller passed
  `may_delay=true`; `ScalarFallback` remains the catch-all for 2×2
  and swap cases.
- **`j_start` policy unified**: for both `ScalarFallback` and `Delayed`
  the caller passes `j_start = k + n_elim + 1` to `apply_blocked_schur`.
  Both statuses have the same peek-ahead-but-no-pivot shape at column
  `k+n_elim`; the only difference is what the caller does next (scalar
  step vs. break).
- **Gate simplification**: `factor_frontal_blocked` now falls back to
  `factor_frontal` only on `bs < 2 || ncol <= bs` (configuration edge
  cases). `may_delay` no longer forces fallback.

## Abandoned Approaches

None this session. Considered and kept:

- **Returning a richer status from the panel** (e.g. carrying the
  rejected-pivot's column index out). Not needed — the caller derives
  it from `n_elim`.
- **Applying the deferred Schur before returning from the panel**.
  Would save one outer-loop call but duplicates state the caller
  already manages (`j_start` decision, break vs continue). Keeping
  the panel stateless w.r.t. the Schur update simplifies the
  invariants.
