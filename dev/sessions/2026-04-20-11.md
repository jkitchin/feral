## Regression Report (MANDATORY — CLAUDE.md)

**Known non-determinism bug in the new parallel driver.** Factorising
the 169 585-matrix KKT corpus via
`factorize_multifrontal_supernodal_parallel` produces ~1-2 % inertia
mismatches vs `factorize_multifrontal` under default (multi-thread)
rayon. `RAYON_NUM_THREADS=1` gives 0 / 38 878 mismatches, so the
algorithm is correct under serial execution.

Rule-outs attempted this session:

| Hypothesis                                 | Experiment                           | Result                 |
|--------------------------------------------|--------------------------------------|------------------------|
| Per-thread workspace lifecycle             | Single `Mutex<FactorWorkspace>`      | Race persists, 5/364   |
| SIMD nondeterminism (pulp)                 | `FORCE_SCALAR_FRONTAL=true`          | Race persists, 5/279   |
| Caller-thread / worker-0 workspace sharing | Extra slot at `num_threads+1`        | Race persists          |
| Rayon pool at all                          | `RAYON_NUM_THREADS=1`                | 0 / 38 878 mismatches  |

Root cause still unidentified. Because CLAUDE.md's hard rule is
"inertia must be exactly correct — no tolerance on inertia counts",
the dispatcher is wired to the sequential path; the parallel function
is kept public for future debugging. Never loosened any test tolerance.
