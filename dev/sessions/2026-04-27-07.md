## Decisions Made

- **`MinFill::run_elimination` stays stubbed in Phase B.1, gated on
  invariant tests.** Considered going straight from B.1 into B.2's
  ~400–600 LoC inner-loop port; rejected because (a) Phase C's MUMPS
  HAMF4 oracle is not yet plumbed so the only validation available
  is the generic permutation invariant, and (b) CLAUDE.md mandates
  tests-first. Landing the invariant tests at session 07 means B.2
  has a passing MinDegree gate plus a clearly-failing MinFill gate
  to drive against.
- **MinFill bucket arithmetic clamps `score ≤ 0` to bucket 0.** The
  research note's RMF/(NVI+1) division should not produce negative
  values in well-formed runs, but defensive clamping is one i32→
  usize conversion and avoids a panic if a future bug ever does
  cause underflow. Documented inline.
- **Tests-first hand-derived fixtures (3×3 hub-last, 5×5 dual-hub
  deferred) are *not* yet pinned to expected perm sequences.** The
  generic permutation invariant is what session 07 ships; the
  tighter expected-perm fixtures land alongside Phase B.2's inner
  loop where they can be cross-checked against a working
  implementation. The hand-derivation-without-oracle path was
  judged too error-prone for autonomous execution.
