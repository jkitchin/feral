## Accomplished
- New module `crates/feral-kahip/src/data_reduction.rs` implementing
  all four Ost-Schulz-Strash 2021 reduction rules:
  - Rule 1 (degree-1) with cascading leaf elimination (LIFO worklist).
  - Rule 2 (degree-2) with both simplicial (`u ∼ w`, zero fill) and
    non-simplicial (`u ≁ w`, one fill edge) sub-cases. Per-pass
    `skip[]` array handles pure-cycle-through-branch (u == w) without
    aborting the whole pass — a real bug discovered during testing.
  - Rule 3 (twins) for both open (`N(u) = N(v)`) and closed
    (`N[u] = N[v]`) forms, using canonical sorted signatures.
  - Rule 4 (neighborhood subset) with mark-array + token counter.
    Conservative scope; more aggressive tuning deferred to K6 bench.
- Anchor-based expansion via path-compressed union-find. Each
  eliminated vertex anchors to a surviving vertex; anchored
  eliminations emit just before their anchor in the expanded

## Decisions Made
- `BTreeSet<usize>` per-vertex adjacency with alive-flag tombstones.
  O(log deg) insert/remove; iteration is sorted. Acceptable for K1
  correctness; revisit at K6 if hot.
- Anchor union-find with path compression for expansion (instead of a
  flat Vec<Vec<usize>> — that was audit-flagged Bug 4). Amortized
  O(n + |ops|) expansion.
- Rule 2 uses a per-pass `skip[]` array and `continue 'outer` on
  cycle-through-branch; previous "break whole pass" semantics would
  have lost legitimate chain compressions.
- Rule 4 scope intentionally conservative — wraps a mark-array
  token counter, iterates all alive vertices once per pass. K6
  benchmarks will drive any further tuning.
- K1 stays internal until K6. No `OrderingMethod::KahipND`.
