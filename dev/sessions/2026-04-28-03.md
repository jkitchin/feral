# Session 2026-04-28-03

## Goal
Phase A2 — implement swap-required 2×2 inline in the deferred-Schur
panel. Plan target: ≥75% drop in `swap` panel-bails (the
2026-04-28 attribution probe showed 82.6% of bails are
swap-required 2×2). Estimated 2-3% wall improvement.

## Accomplished

- **Phase A2 c==0 inline path landed** (commit `dfe169e`). The
  panel now handles swap-required 2×2 (`r > col + 1`) inline at
  the FIRST pivot of any panel, where peek-ahead is a no-op and
  the deferred state IS the scalar state. Mid-panel (`c > 0`)
  swap-2×2 still bails to scalar.

- **Threading + counter** (part of `dfe169e`): `lblt_panel_frontal`
  takes `perm: &mut [usize]`; new `INLINE_2X2_SWAP_OK` counter in
  `panel_diag` module; `probe_panel_attribution` prints `swap_ok`
  alongside swap bail counts.

- **Test fixtures byte-identity** (committed `9a576ef` last
  session): three new fixtures pass byte-identical against scalar:
  `test_swap_2x2_inside_panel_bare` (under `block_size: 8`
  forcing the swap to land at `c == 0`), `test_swap_2x2_chain`,
  `test_swap_2x2_then_1x1`. The bare test asserts
  `inline_2x2_swap_ok >= 1` to verify the inline path is actually
  taken; the chain and then-1×1 tests still exercise the c > 0
  ScalarFallback path under the default `block_size: 64` (the
  swap is mid-panel) — they verify byte-identity on bail-after-swap.

- **Regression caught and fixed**: an initial draft missed
  scalar's `r_is_fully_summed = r < ncol` precondition for any
  swap pivot (`scalar_pivot_step:2120,2167`). Without it, ACOPR30
  iterate 0067 produced inertia (398, 166) vs reference
  (400, 164) — diff of 2. Adding `(!need_swap || r < ncol)` to
  the panel's `bounds_ok` check restored byte-identity. All 208
  tests pass after the fix.

- **Honest measurement**: re-ran `probe_panel_attribution` —
  `swap_ok = 0` aggregate. ALL corpus swap-2×2 cases happen at
  `c > 0`, so the c==0 restriction catches **none** of them. Swap
  bails dropped 19 → 18 (a single case from the new `r < ncol`
  gate matching scalar more precisely, NOT from inline accept).
  **The plan's ≥75% bail-drop acceptance criterion is NOT met.**

## Decisions Made

- **A2 c==0 restriction**: ship the safe c==0 subset rather than
  block on the mid-panel infrastructure. Rationale: c==0 is
  bit-exact without any new replay primitives, lays the API
  groundwork (perm threading, counters, probe output, fixture
  patterns), and the regression risk is low. Mid-panel extension
  becomes a focused follow-up plan.

- **Mid-panel swap-2×2 deferred**: the correct mid-panel
  implementation needs (1) `peek_ahead_replay` called twice
  (col+1 AND r), (2) a row-r-left-of-diagonal replay primitive
  for the entries `symmetric_row_offdiag_max` reads, (3) a new
  `apply_blocked_schur(..., skip_col=Some(r))` parameter to
  avoid double-applying when bail-after-peek leaves r in scalar
  state, and (4) a `ScalarFallbackPeekedTwo{col1, r}` variant
  for bail-state semantics. Estimated 4-6 hours.

## Abandoned Approaches

None this session. The c==0 restriction is intentional scope
narrowing, not abandonment — the mid-panel work is still on the
roadmap as a separate follow-up plan.
