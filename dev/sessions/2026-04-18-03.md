Changes:
- `src/symbolic/mod.rs`: removed `use crate::ordering::amd::amd_order`

  minimum-degree column 0 by adding `(2,1) = 0.5`. Root cause of the

  amalgamation merged the tiny-pivot column into the root supernode,
  collapsing the "delays propagate up" test. The new fixture still
  exercises the delayed-pivoting path (2 supernodes: {col 0},
  {cols 1,2,3}; tiny pivot at leaf; delay propagates to root) and is
  strictly more robust against AMD tie-breaking.

## Decisions Made

- **Retire in-tree AMD** (`src/ordering/amd.rs`) in favor of
  `feral-amd`. Rationale: strictly better on larger matrices (fill
  and wall-clock), statistically tied on the parity corpus,
  reduces maintenance surface. Recorded in `dev/decisions.md`.
- **Scaffold KaHIP rather than alias METIS.** The four-ordering
  story (AMD, METIS, SCOTCH, KaHIP) needed a public crate name and
  API surface reserved; publishing a stub that returns `Internal`
  is honest about capability, whereas aliasing METIS would mislead
  downstream consumers. Dispatch wiring deferred to phase K6.
