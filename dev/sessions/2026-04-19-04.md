## Accomplished

### D.1 FactorWorkspace (commits `9c0419b` → `dedb3f3`)

- Added `FactorWorkspace` struct with 5 scratch fields: `row_map`,
  `frontal_values`, `build_delayed`, `build_trailing`, `build_seen`.
  Each grows monotonically; all are cleared-then-resized per call
  so cross-call state can't leak.
- `factorize_multifrontal_with_workspace` now draws from `ws`
  instead of allocating per-supernode. `std::mem::take` + clear +
  resize pool pattern for the frontal buffer and row-index scratch.
- Rewrote `build_row_indices`: `BTreeSet<usize>` dedup → boolean
  `seen` array + sort. O(1) insert, O(m log m) final sort.
- `Solver::factor()` reuses a workspace across calls. IPM consumers
  amortise heap traffic across iterations.

## Decisions Made

- **D.3 thresholds finalized:** `N_MAX = 128`, `ρ_MIN = 0.25`.
  Justified by the stage-2 synthetic sweep: at ρ = 0.25 the dense
  path wins for every swept n, with 0.49–0.66× margin; the safety
  factor covers real IPM KKTs that factor 1.5–2× slower than
  diagonally-dominant synthetics. Appended to `dev/decisions.md`.
- **D.1 exited at geomean 0.46** rather than extending into
  narrower workspace-slice optimisations. Per the D.1 plan's
  ex-ante exit clause, this was the right off-ramp — the remaining
  tail was structural (arrow-KKT, tiny-n, small-dense), not alloc
  churn.
- **Delayed-pivot tests now call `factorize_multifrontal_supernodal`.**
  The `_supernodal`/`_supernodal_with_workspace` entry points are
  now part of the public API as the documented "gate bypass" for
  tests that need to exercise the multifrontal body on in-gate
  matrices.
