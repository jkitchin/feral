## Decisions Made

No new architectural decisions this session. All choices were
within the M4–M7 plan:

- **Lazy `BinaryHeap` priority queue for FM**, not METIS's bucket
  array. Captured in the fm_refine.rs module docstring; adequate
  at FERAL's target scale (≤ 100k vertices, FM rarely dominates).
- **Kuhn augmenting-path matching**, not Hopcroft-Karp. Documented
  in separator.rs; `O(V·E)` on boundary subgraphs at `n ≤ 120` is
  cheap. Hopcroft-Karp can replace it later if boundaries grow.
- **Greedy separator refinement**, not two-sided FM. Documented in
  fm_refine.rs. Full FM deferred until a concrete quality gap
  motivates it.
- **AMD leaf fallback via feral-amd path dep**, not a rewritten
  inline minimum-degree kernel. No crate cycle; feral-amd already
  only depends on feral-ordering-core.

None of the above cross the "architectural decision" bar, so
`dev/decisions.md` is unchanged.
