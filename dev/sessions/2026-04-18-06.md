## Decisions Made
- **Fixed-node pinning with fallback.** Plan/research say "all
  band vertices at distance EXACTLY `bnd_distance`". For small
  parts fully inside the BFS ball, no such vertex exists.
  Adjusted rule: `pin_depth_p = min(max_dist_in_part, bnd_dist)`
  per side, then pin all part-p vertices at dist == pin_depth_p.
  Preserves the "all deepest" semantics while handling the
  degenerate case.
- **INF_CAP = (sum_band_edge_weight / 2) + 1.** Using
  `i64::MAX / 4` overflowed `excess[source]` when many vertices
  pin with that capacity. The sum-based bound is tight: in-band
  cut is bounded by total in-band edge weight, so INF_CAP just
  exceeds it. Halved because the edges array counts each
  undirected edge as two anti-parallel directed entries.
- **Two-cut MBMC v1 via second solve.** Rather than extract
  sink-closest cut via residual-graph BFS from sink, run a
  second `push_relabel` call with s/t swapped and edges
  reversed. Clumsier but reuses the primitive verbatim; full
  MBMC is K5/K6 follow-up.
