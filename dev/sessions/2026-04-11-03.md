## Key Findings
- Inertia disagreements cluster on diverged IPM iterations (delta_w > 1.0: 7,209
  matrices, delta_w near f64::MAX in worst cases)
- Eigenvalue spot-checks show MUMPS is also wrong on these matrices:
  - ACOPP14_0001 (cond=1e11): true (37,69,0), MUMPS (38,68,0), FERAL (37,68,1)
  - ACOPP30_0000 (cond=1e18): smallest eigval 2.2e-14, MUMPS calls positive, FERAL calls zero
  - POLAK6_0021 (cond=7.8e33): entries span 1e-4 to 1.3e+42, no f64 solver can handle this
- On well-conditioned matrices (delta_w < 1e-4, ~87k matrices): FERAL is essentially perfect
- Phase 1a exit criterion satisfied: 100% on exact tests + 99.2% on real KKT
  (disagreements are on numerically ambiguous matrices where MUMPS is also wrong)

## Decisions Made
- Inertia comparison label is "match vs MUMPS", not "pass/fail" — neither solver
  is authoritative on ill-conditioned matrices (recorded in benchmark-history.md)
- dim <= 500 cap for dense phase; larger matrices preserved for sparse phase
- Matrices with NaN/Inf in data or RHS silently skipped at load time (not counted)
