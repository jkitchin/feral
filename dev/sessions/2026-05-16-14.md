## Abandoned Approaches

None this session. Two latent findings worth flagging though
(both documented in the research note §5 / §6):

- **FMA path regression** on aarch64: `BunchKaufmanParams::fma=true`
  is 0.65–0.93× speedup vs no-fma on the probed shapes. Contradicts
  issue #8's expected 2× speedup. Worth a separate investigation but
  was not in scope for issue #14.
- **`MAX_N_ELIM = 64` latent panic** at `src/dense/factor.rs:2384`:
  raising `BunchKaufmanParams::block_size` above 64 (e.g. via
  `PROBE_BLOCK_SIZE=128`) panics with a stack-buffer overflow. Fix
  is trivial; not exploited because default is 64 and the panel
  clamps `bs.min(ncol)` for ncol < 64.

## Next Session Should

In priority order (mirrors §7 of the research note):

1. **Driver-overhead probe on MBndryCntrl_3D_27.** Sequential factor
   reports 2.45 s, sum of measured kernel time on the dominant
   supernodes is ~1.5 s → ~1 s of driver overhead unaccounted for.
   Rerun `diag_supernode_cost` / `Profiler` on the 3608-supernode
   build and confirm where it lands. If driver is dominant, the
   lever is issue #13 pooling extensions, not kernel work.
2. **Symbolic / amalgamation A/B vs MA57** on MBndryCntrl_3D_27.
   The sum of top-2 supernode kernel times (311 ms) already exceeds
   MA57's full factor (330 ms), so MA57 must be producing a
   fundamentally different supernode tree. Run METIS-ND + nemin
   sweep and compare supernodes.len() / max(nrow) / fill ratio vs
   MA57's reported numbers.
3. **Update GitHub issue #14** with the findings from
   `dev/research/wide-supernode-throughput-2026-05-16.md` so the
   issue's premise is corrected and the next reader doesn't start
   the kernel rewrite based on the 1.3 GFLOPS headline.
4. **FMA path regression investigation**: direct microbench of
   `schur_panel_minus_fma_strided_quad` vs the nofma sibling. If
   the FMA path is genuinely slower, the production dispatch could
   be simplified.
5. **`MAX_N_ELIM` fix**: raise to 128 (+4 KB stack/panel) or clamp
   `bs.min(MAX_N_ELIM)` in the entry point. Either eliminates the
   latent panic.
