## Decisions Made
- AMD ordering uses exact external degree (not approximate) — sufficient for
  Phase 1b matrix sizes (dim <= 500). Full AMD with element absorption is
  Phase 2 optimization.
- Column counts via elimination graph simulation rather than Liu's row subtree
  algorithm — simpler, correct, O(n^2) worst case but fine for dim <= 500.
- Sparse solve uses dense_solve per supernode (gather/scatter) rather than
  custom supernodal forward/backward substitution — correct and simple.
- nemin default = 32 matching SSIDS.
