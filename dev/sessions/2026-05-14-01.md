### F2.3 implementation

- `src/numeric/solve.rs`: refactored `solve_sparse_refined` into a
  private `solve_sparse_refined_core(..., with_diagnostics: bool)`
  shared by both public surfaces. Control flow and returned iterate
  are bit-identical between `solve_sparse_refined` and
  `solve_sparse_refined_with_diagnostics` (test
  `diagnostics_match_non_diagnostic_solution` verifies
  `f64::to_bits` equality on a KKT 3×3).
- New public types `RefinementStep` and `RefinementDiagnostics`:
  - `RefinementStep { step, residual_2norm, relative_residual,
    forward_error_bound, improved }`.
  - `RefinementDiagnostics { anorm_1, kappa_1_est, steps,
    returned_step }`.
  - `kappa_1_est` is computed **once** per refinement run (3-5
    extra solves) — it's a property of (A, factor), not of x.
    Per-step `forward_error_bound = kappa_1_est * relative_residual[k]`
    derives the Skeel 1980 / Higham 2002 §15 conservative upper
    bound on `||x - x_true||_∞ / ||x_true||_∞`.
- Re-exported from `src/lib.rs` for downstream consumers
  (ripopt's δ-ladder logging).
- Six new tests in `src/numeric/solve.rs::tests`:
  1. `diagnostics_match_non_diagnostic_solution`
  2. `diagnostics_populate_well_conditioned`
  3. `diagnostics_kappa_matches_standalone` (bit-exact equality
      between `diag.kappa_1_est` and the standalone
      `estimate_condition_1norm` on Hilbert n=6)
  4. `diagnostics_forward_error_bound_field` (identity check)
  5. `diagnostics_n_zero`
  6. `diagnostics_dim_mismatch_rejected`

  Evidence: `cargo test --lib`: 246 passed, 0 failed, 5 ignored.
  `cargo clippy --all-targets -- -D warnings`: clean.

### F2.2 corpus parity audit (`src/bin/diag_cond_parity.rs`)

- **Confirmed directional, not parity-gated.** The binary
  computes `kappa_feral / cond2` and `kappa_feral / max(cond1,
  cond2)` statistics (count, geomean, min/p10/median/p90/max,
  worst+best |ln(ratio)|) and exits 0 regardless. No assertion
  gates the corpus on a tight band — appropriate given the
  operator mismatch (MUMPS `RINFOG(10)/(11)` are componentwise
  ∞-norm condition numbers; feral computes the 1-norm form).
- **Minor staleness:** the header docstring mentions a wider
  "[0.1, 10.0] band" treated as a directional check. The plan's
  updated empirical observation is geomean ≈ 4.244e10 over 165959
  corpus matrices (see 2026-04-27-09 journal entry). The "[0.1,
  10.0]" framing is overoptimistic by ~10 orders of magnitude
  but harmless because no enforcement exists. Worth a
  one-line clarification in a follow-up; not blocking.

## Decisions Made

- **κ̂ is computed once per refinement run, not per step.** The
  research note (`dev/research/condition-estimate.md`) treats κ̂
  as a property of `(A, factor)`. Recomputing it per step would
  multiply the cost by the refinement-iteration count (up to 10×)
  for zero new information. What varies per step is the
  forward-error *bound* `κ̂·η_rel`, which is cheap to derive.
  Recorded in journal `2026-05-14-01.org` (entry 09:55).
- **`with_diagnostics` is opt-in.** The default
  `solve_sparse_refined` is unchanged — it goes through the core
  with `with_diagnostics: false`, skipping the κ̂ and the
  per-step `Vec` allocation. No bench regression expected.
- **Diagnostic struct kept owned (`Vec<RefinementStep>`)** rather
  than callback-driven. Callers who want per-step emission can
  iterate the returned vector; callers who want only summary data
  (worst residual, final forward-error bound) read scalar fields.
  This matches MUMPS's `RINFOG`/`INFOG` postmortem-array shape
  rather than a streaming callback.
