### 2. Scaling benchmark — `external_benchmarks/scaling/`

New harness with four files (plus generated artifacts):

- `run.py` (325 lines) — generates synthetic matrices for four
  families, writes RHS as `b = A·x_true`, builds per-solver

- `.gitignore` for the regenerable `matrices/`, `rhs/`, `out/`
  directories (~177 MB of synthetic data).

### 4. Report findings

The data sorts cleanly. Three actionable items:

1. **Symbolic analysis dominates on regular sparse families.**
   feral's analyse-fraction at the largest sampled n:
   `banded_spd` 0.96 (n=262144), `laplace2d` 0.91 (n=36864).
   On `laplace2d`, feral's *numeric factor* is 10× faster
   than MUMPS (12.9 ms vs 138.8 ms) — the visible "factor
   time" gap is entirely symbolic-analysis overhead.
2. **Banded structure unrecognized.** MA57 dominates banded
   SPD by 5–20× (80 ms vs feral 1.88 s at n=262144) by
   exploiting band locality through pivoting.
3. **Dense kernel needs blocking work.** feral/MUMPS ratio on
   `dense_si` grows from 0.81 (n=64) to 2.21 (n=1024).

Two fundamental constraints visible in the data: single-threaded
execution (vs MUMPS OpenMP) and the pure-Rust constraint
ceiling on dense kernel performance.

Important refinement from the data: on `saddle_kkt` (feral's
intended NLP/KKT workload) the analyse-fraction is only 0.12 at
n=2048 — factor dominates there. So the "fix symbolic analysis"
guidance is family-dependent: highest leverage for PDE-flavoured
sparse, less so for KKT.

## Decisions Made

- **Scaling bench lives at `external_benchmarks/scaling/`** as a
  Python driver, not as a new Rust bin. Reuses the existing
  per-matrix solver drivers (`bench_one_matrix`, `mumps_bench`,
  `ma57_bench`) and keeps matrix-generation logic flexible.
- **`total_factor_us = analyse_us + factor_us`** is the
  comparison metric for cross-solver plots. MUMPS bundles
  analyse and factor under `JOB=4` so its `factor_us` already
  contains both phases; feral and MA57 report them separately.
  Without this normalization, comparisons are apples-to-oranges
  (initial laplace2d plot suggested feral was 11× faster than
  MUMPS at n=36864 — actually parity once analyse is included).
  Documented in `plot.py:load` and the report's Methodology.
- **Generated matrices (~177 MB) are gitignored.** They are
  fully reproducible from `run.py` with a fixed seed.
- **`assemble-context.sh` default skips the corpus bench.** The
  bench section is sourced from the latest dated session
  checkpoint. Pass `--with-bench` for fresh numbers.
