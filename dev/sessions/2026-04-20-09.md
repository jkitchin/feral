# Session 2026-04-20-09

## Goal

Phase 2.5.1 — replace the `O(n²)` `column_counts` in
`src/symbolic/column_counts.rs` with the Gilbert–Ng–Peyton
`O(nnz(A) + n·α(n))` algorithm using Liu's row-subtree
characterization. Phase 2 planning doc's highest-leverage
sparse-pipeline item.

### Implementation

- `src/ordering/elimination_tree.rs`
  - `postorder()` — iterative DFS with explicit stack.
  - `first_descendants(&post)` — smallest postorder number
    across each node's subtree, used for the GNP leaf test.
  - 5 new unit tests (chain, star, two-roots, first-chain,
    first-leaves). Etree module: 5 → 10 tests.

- `src/symbolic/column_counts.rs`
  - `column_counts_gnp(pattern, etree)` with inline
    path-compressed DSU for LCA queries.
  - First draft failed 5/6 parity tests. Root cause:
    (a) had partner/current variable roles swapped in the leaf
        test — should be `first[current] > maxfirst[partner]`
        with `partner > current`.
    (b) missing the head-of-loop `delta[parent[j]] -= 1` for
        non-root j — cancels the double-count at accumulation.
  - Corrected: 6 GNP parity tests pass bit-exact against the
    reference.

- `src/symbolic/mod.rs:334` — production call site switched from
  `column_counts` to `column_counts_gnp`. Old function kept
  public as a reference oracle for existing supernode tests.
