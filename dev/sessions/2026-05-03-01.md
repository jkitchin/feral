## Goal

Diagnose and fix the structural inflation of `factor_nnz` (19× over
textbook L-fill on PoissonControl K=158) that was causing the
~650× factor-time gap vs MUMPS reported in the prior session. User
asked for a detailed plan with tests demonstrating the fix without
compromising accuracy.

After the build_row_indices fix landed, the session extended to:

## Accomplished

### `build_row_indices` upper-triangle pollution fix (commit `9b2665f`)

- Identified root cause: `build_row_indices` at
  `src/numeric/factorize.rs:2241-2308` iterates the fully-symmetrized
  A pattern (`full_pattern = matrix.symmetric_pattern()`,
  `csc.rs:186-239`) and absorbed both lower-tri (r > j, legitimate
  trailing) and upper-tri (r < j, already-eliminated) rows into every
  supernode's frontal. Symbolic side filters correctly
  (`column_counts_gnp` skips `partner <= i`); only numeric was
  over-collecting. Pollution propagated up the etree through child
  contrib blocks.
- Added `cfg(debug_assertions)` invariant assertion at
  `src/numeric/factorize.rs:1469-1485` enforcing that every trailing
  row is `>= first_col + own_ncol`. The assertion failed on 6 existing
  tests, pinning the bug's reach (Schur-block tests, multi-supernode
  Schur, summary one-liner).
- Filter `r < first_col + own_ncol` in both the native pattern loop
  and the child contrib loop in `build_row_indices`
  (`src/numeric/factorize.rs:2274-2298`).
- Wrote `tests/build_row_indices_trailing_invariant.rs` (8 tests, all
  fixtures with n > N_TINY=16 to exercise the multifrontal path).
- Verified all 216 lib + integration tests pass after the fix.

### Phase A — `SupernodeParams::default().nemin` 32 → 16 (commit `4c0fc80`)

- Diagnosed the post-fix 1.6-2× residual gap between numeric
  `factor_nnz` and `Σ col_counts` as dominantly **pass-through row
  padding** (children's contribs flowing through ancestors that
  don't pivot on those rows, stored as zeros in the dense trailing
  rectangle). NOT delayed-pivot fill — K=50 has 0 delayed pivots
  yet shows 2.47× inflation. Wrote
  `dev/research/factor-nnz-residual-gap.md`.
- Swept `nemin ∈ {8, 16, 32, 64}` × {AMD, METIS-ND} on PoissonControl
  K=50 and K=158. `nemin = 16` is the sweet spot:
  - K=50 AMD: factor_nnz -26%, factor wall -18%
  - K=158 AMD: factor_nnz -21%, factor wall ≈ par
  - `nemin = 8` recovers more memory but regresses K=158 wall +21%
- Flipped `SupernodeParams::default().nemin` from 32 to 16.
- Relaxed `nrow_matches_symbolic_*` → `nrow_at_least_symbolic_*`
  in `tests/build_row_indices_trailing_invariant.rs` — the previous
  `assert_eq!` was conceptually wrong (conflated `Supernode.nrow`
  with the working frontal nrow); only happened to hold at
  `nemin=32` on small fixtures with no pass-through padding.
- Corpus bench unchanged at PASS gates: dense small-frontal P90 1.34
  (≤ 2.0 PASS), medium 1.70 (≤ 3.0 PASS); sparse 1.56 / 1.57 PASS.

### Issue #2 — `NumericParams::default()` `pivot_threshold = 1e-8` (commit `3b4f248`)

- Replaced `#[derive(Default)]` on `NumericParams` with a manual
  `impl Default` that sets `bk.pivot_threshold = 1e-8` (matching
  MA27's `cntl[1]` / Ipopt's `ma27_pivtol`).
- Root cause: ripopt and other consumers building
  `NumericParams::default()` were inheriting `0.0` from
  `BunchKaufmanParams::default()`, silently disabling the
  column-relative pivot rejection, rook rescue, and delayed-pivoting
  cascade. On rank-deficient KKT-augmented LS-init systems
  (`A = [I J^T; J diag]`, m > n, equality rows D=0; e.g.
  CUTEst arki0003) this caused 58 zero `y_d` entries clustered at
  `_scon[2052..2138]`.
- Why 1e-8 and not the SSIDS/MUMPS canonical 0.01: 0.01 is validated
  on MC64-equilibrated matrices; ripopt runs Identity scaling, where
  0.01 over-rejects. 1e-8 is what Ipopt+MA27 ships with for this
  exact unscaled-KKT configuration. In-tree MC64-scaled callers
  keep their explicit 0.01 override.
- `BunchKaufmanParams::default()` (dense) intentionally stays at
  0.0 per the 2026-04-13 dense/sparse split. `NumericParams::with_bk`
  takes its `bk` argument verbatim — no override applied.
- `Solver::increase_quality` cascade docstring updated; W5
  "0.0 → 0.01" first-jump rule preserved for callers that
  explicitly disable the threshold.
- New regression test `tests/issue_2_kkt_ls_init.rs` (3 tests):
  asserts dense default unchanged, sparse default = 1e-8, and a
  rank-deficient KKT-augmented LS-init repro that produces no
  exact-zero multiplier rows under the new default.
