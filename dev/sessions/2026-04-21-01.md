# Session 2026-04-21-01

## Goal
Root-cause and fix the ~1–2% non-deterministic inertia mismatch in the Phase 2.5.2 Rayon parallel multifrontal driver that session 2026-04-20-11 landed gated-off, then re-enable parallel dispatch.

## Accomplished
- **Root-caused the parallel race.** The seeding loop in
  `factorize_multifrontal_supernodal_parallel` used
  `pending[leaf_idx].load() == 0` as the leaf predicate. Inside
  `rayon::scope`, workers start executing spawned leaves immediately
  and decrement their parents' `pending` counters concurrently with
  the seed loop. A parent whose final child completed mid-seed
  would read `pending == 0` in the caller AND get trampoline-spawned
  by its last child via `fetch_sub==1` — same supernode factored
  twice, a different supernode never dispatched.
- **Evidence.** `FERAL_HASH_FRONTAL` instrumentation in
  `factor_one_supernode` logged the pre-factor frontal hash for
  every call. `diag_par_frontal_hash` looped ACOPR14_0003 until
  divergence (attempt 68), then diffed the per-run hash streams.
  Run A had snode 9 once + snode 173 (root) once; run B had snode
  9 TWICE and snode 173 ZERO times. Proof of double-dispatch, not
  dense-kernel nondeterminism.
- **Fix.** Replaced dynamic `pending==0` predicate with a static
  "supernode has no children" filter computed before entering the
  scope. `src/numeric/factorize.rs:929-961`.
- **Validated.** `diag_par_repeat` across 38 878 corpus runs:
  par-vs-par-nondet = 0, par-vs-seq-mismatch = 0.
  `diag_par_frontal_hash` 200 attempts on ACOPR14_0003: no
  divergence. All 6 `parallel_parity` tests pass with
  `#[ignore]` removed. Full `cargo test --release`: 251 tests pass,
  1 ignored (unrelated), 0 fail. `cargo clippy -- -D warnings`
  clean.
- **Re-enabled dispatcher.** `factorize_multifrontal_parallel_with_workspace`
  now routes to the parallel driver when `should_parallelize_assembly`
  is true. Removed the "KNOWN BUG" doc blocks on the predicate, the
  driver, and the dispatcher.
