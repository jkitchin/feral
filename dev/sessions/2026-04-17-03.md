### New crate `crates/feral-scotch`

- `Cargo.toml` — single dependency on `feral-ordering-core` (path).
  Registered in workspace `Cargo.toml` members list.
- `src/lib.rs` — defines `ScotchOptions` and `ScotchStats` with
  defaults matching the audited plan
  (`compress_ratio = 0.7`, `amd_switch = 120`,
  `coarsen_floor = 100`, `n_sep_trials = 5`, `fm_move_cap = 200`,
  `fm_pass_cap = 32`, `max_imbalance = 0.05`,
  `seed = 0xDEAD_BEEF`). The full-pipeline contract producer
  `scotch_order` is *not* yet exposed — it lands in S5 once the
  separator + driver are in.
- `src/graph.rs` — CSR `Graph` + `from_csc_pattern`. Near-mirror of
  `feral_metis::graph` (intentional duplication for S1 standalone-
  ness; will be unified or shared via a future `feral_metis::
  internals` exposure when S2 needs the coarsening / FM
  infrastructure).
- `src/test_util.rs` — shared `csc_from_edges` test helper, gated
  on `#[cfg(test)]`.
- `src/compress.rs` — the S1 deliverable. Algorithm:
  1. Per-vertex closed-neighborhood hash via
     `std::collections::hash_map::DefaultHasher` (SipHash-1-3,
     deterministic — *not* `RandomState`).
  2. Hash-bucket assembly.
  3. Equivalence-class assignment via exact closed-neighborhood
     compare within each bucket; representatives chosen in
     ascending original-index order for determinism.
  4. CSR re-assembly with **summed vertex weights** *and*
     **summed edge weights** across parallel edges that collapse
     onto the same compressed pair. Self-loops (intra-class
     edges) dropped.
  Returns `Some(CompressedGraph)` only when
  `n_compressed / n_original < max_ratio` (strict less-than).

Highlights:

- **Edge-weight summing.** `parallel_edges_sum_their_weights_on_merge`
  builds two `K_3` blocks bridged by a single edge; vertices `{0,1}`
  merge into supervariable A, vertex `2` is alone (B), vertex `3`
  alone (C), `{4,5}` merge into D. Asserts the compressed adjacency
  is `A→B(2), B→A(2), B→C(1), C→B(1), C→D(2), D→C(2)` — confirming
  that two original edges `(0,2)` and `(1,2)` collapse to a single
  compressed edge of weight 2. This is the bug the audit (`dev/
  plans/ordering-scotch.md` finding 5) explicitly warned about.
- **Negative case (4×4 grid).** Verifies the algorithm does not
  over-merge — every grid vertex has a unique adjacency set.
- **Conservation.** Vertex-weight sum and external-edge-weight sum
  are preserved up to the dropped self-loops.

## Decisions Made

No new architectural decisions this session — all choices fit
within the audited plan.

Documented in the research note (not in `dev/decisions.md` because
they are scoped to S1 only):

- **Closed-neighborhood-only compression.** Case-1
  indistinguishables (non-adjacent pairs with `N(u) == N(v)`) are
  not detected by the closed-neighborhood hash. Two cases are
  rare in mesh / KKT workloads; revisit if a real corpus reveals
  benefit. Documented explicitly in `compress.rs` module
  docstring and the research note.
- **Hash determinism via `DefaultHasher`.** SipHash-1-3 with
  fixed seeds — deterministic across runs and platforms.
  Explicitly *not* `RandomState`, which is non-deterministic.
- **Standalone graph type for S1.** `crates/feral-scotch/src/
  graph.rs` is a near-mirror of `feral-metis`'s. Will be lifted
  into a shared module (most likely `feral_metis::internals` per
  the user's earlier decision) when S2 needs the coarsen / FM
  infrastructure.

None of the above cross the "architectural decision" bar, so
`dev/decisions.md` is unchanged.
