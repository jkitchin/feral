### 2. Wired `solve_many_refined` into the C ABI

`src/capi.rs:feral_solve` now routes through
`Solver::solve_many_refined` against the cached `s.matrix`. Opt-
out via `FERAL_REFINE=0|false|off|no`. Added two inline unit tests
exercising both paths on the 2x2 indefinite `[[1,2],[2,1]]` (#17
forensics canonical example).

Justification for using `s.matrix` as the refinement reference:
Ipopt's `MultiSolve` protocol guarantees no values writes between
`feral_factor` and `feral_solve` — Ipopt only writes via the
`feral_values_ptr` returned pointer, and only between successive
`MultiSolve` invocations. So `s.matrix` at solve time equals the
matrix that was factored.

Test results: `cargo test --lib --release` → **248 passed** (was
246 + 2 new), 0 failed, 5 ignored. `cargo clippy --all-targets
--release -- -D warnings` clean. `cargo fmt --check` clean.

## Decisions Made

- **Iterative refinement (`solve_many_refined`) is the
  downstream fix for the cascade-break L-factor perturbation.**
  Make it the default for both IPM consumers (feral C ABI,
  pounce-feral). Keep cascade-break enabled — it helps on
  calibrated matrices per feral#8 and #15, and the perturbation
  it introduces is absorbed by one round of refinement against
  the original matrix. Append to `dev/decisions.md`.
