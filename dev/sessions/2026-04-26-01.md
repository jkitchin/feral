## Accomplished

**Mittelmann correctness gate** (commits 779baa9, affee0c, 9d424af,
b2627f2, 7d22ee3): `diag_mittelmann.rs` now loads sidecar inertia +
RHS, solves with `solve_sparse_refined`, and gates inertia-exact
+ residual ≤ `n·eps·1e6`. `FERAL_DIAG_MAX_N` (default 50000) and
`DIAG_VERBOSE`. Final run: 460 OK / 0 inertia fail / 0 residual fail.

**QCQP drill**: the two original failures (`qcqp500-3nc_0000`,
`qcqp750-2nc_0000`) factored with exact inertia; only iterative
refinement made the difference. Switched to `solve_sparse_refined`
(what production calls anyway). Conclusion logged in journal.

**Oracle factor_nnz plumbing** (9d424af): MUMPS `INFOG(9)` and SSIDS
`inform%num_factor` flow into canonical sidecars as `factor_nnz`.
Bench's `print_perf_comparison` adds `nnzL/MUMPS` and `nnzL/SSIDS`
ratio rows.

**AMD off-by-one bug fix** (commit aa0d097): `create_element` returned
`pme2 as usize` where `pme2: i32` was `-1` when every neighbour of
the pivot had been absorbed in the in-place branch. The cast wrapped
to `usize::MAX` and `finalize_step`'s `pme1..=pme2_incl` loops
panicked with `index out of bounds: the len is 1697 but the index
is 1697`, with RSS spiking to 30 GB before the abort. Fix: switch
the entire pipeline to exclusive-end semantics (`pme1..pme2_excl`).
Validated on `ARWHDNE_0000` reproducer + 41 AMD tests + 162 lib
tests + full ARWHDNE/ subdir parity smoke.

**Bench load-time OOM fix** (202c5f5): `KktEntry.matrix` becomes
`Option<SymmetricMatrix>`; `to_dense()` only fires when
`csc.n <= FERAL_DENSE_MAX`. Sparse loop is unchanged.

**Oracle guards** (ff24741): `--max-n` (default 50000) peeks at the
MatrixMarket header before manifest entry; `--mem-gb` applies
`RLIMIT_AS` to the bench subprocess (Linux-only — macOS does not
honor it). Skip-list mechanism for the SSIDS SIGSEGV pair
(`MSQRTA_0001`, `MSQRTB_0001`).

**Streaming three-ordering parity diagnostic** (4757fdb):
`src/bin/diag_fill_parity.rs` factors each matrix three times under
`{Auto, Amd, MetisND}` so MUMPS (AMD-flavored default) and SSIDS
(METIS default) are each compared against a structurally-equivalent
feral configuration. `params_kkt_sparse` numerics. `FERAL_DIAG_MAX_N`
gate. `catch_unwind` per factor for batch resilience.
