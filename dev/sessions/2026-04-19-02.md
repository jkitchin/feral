## Decisions Made

- `WrongInertia` keeps the factor stored (Ipopt parity). Encoded in
  `FactorStatus` semantics and tested by `solve_after_wrong_inertia_still_works`.
- `Singular` clears the factor (`solve()` after Singular returns
  `FeralError::NoFactor`). Tested by
  `solve_after_singular_returns_no_factor`.
- `PatternFingerprint = (n, col_ptr.len(), row_idx.len())` — collisions
  are tolerated for the IPM use case; tighter hash is a future
  refinement only if a downstream consumer needs it.
- `Solver` is `!Sync` by virtue of the mutable cache state; no parallel
  factor / solve. Callers needing parallel solves must use `factors()`.
- `pivtol_max = 0.5`, `exponent = 0.75` (MA27 defaults) — conservative
  vs MUMPS's 0.5; revisit when POUNCE accumulates evidence.
