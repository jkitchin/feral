## Accomplished

### Phase B-1 (NR=2): dual-column DSYRK micro-kernel — landed

The plan called for a register-blocked MR×NR micro-kernel; this
session lands the narrowest non-trivial NR=2 variant as a stepping
stone. The key transformation it captures: dst0[1..] (column j rows
[j+1, nrow)) and dst1[..] (column j+1 rows [j+1, nrow)) reference
the **same** src memory at column offset `(k+q)*nrow + j + 1`. A
single src load per chunk per q now feeds both accumulator stacks,
halving src memory traffic on the bulk of the trailing update.

**File-level changes:**

- `src/dense/schur_kernel.rs` (+~290 LoC): new
  `schur_panel_minus_nofma_strided_dual(dst0, dst1, src_block,
  src_first_col, n_elim, col_stride, src_row_offset, alphas0,
  alphas1)`. Cap (dst0[0]) processed by scalar prologue; bulk
  (dst0[1..] and dst1) processed in a 4-way unrolled SIMD body
  holding 8 accumulator registers (4 dst0 + 4 dst1). Per-q does one
  shared src load then independent `mul → sub` chains per
  accumulator. Bit-exact per column with the rank-1 reference.
- `src/dense/factor.rs` `apply_blocked_schur_panel`: outer loop
  walks trailing columns in pairs and dispatches the dual kernel.
  `split_at_mut` sequence carves disjoint `dst0 = &mut col_j[j..]`
  and `dst1 = &mut after_j[(j+1)..nrow]` from a single src
  `before` slice. Odd-tail column falls back to the single-column
  kernel.

**Bit-exactness contract:** for each column independently the
per-element rounding sequence
```
acc <- round(acc - round(alpha_q * src_q[i]))
```
is issued in q ascending order, identical to two sequential calls
of `schur_panel_minus_nofma_strided` on columns j and j+1. The
shared src load is a memory-traffic optimization, not a numeric
reordering.

## Decisions Made

- **NR=2 (dual-column) before wider tiles.** The plan's MR×NR
  micro-kernel design space is large; a 1×2 (NR=2) variant captures
  the key src-load-sharing benefit with minimal register pressure
  (8 dst accumulators + 4 src vectors fit easily in 16 AVX2 / 32 NEON
  registers). Wider NR (4, 8) and MR>1 add complexity and may be
  beneficial only on much wider supernodes. Defer wider tiles until
  bench data motivates them.

- **Cap entry as scalar prologue.** `dst0[0]` is the column-j
  diagonal; processing it in a scalar q-loop before the SIMD body
  lets `dst0_bulk = &mut dst0[1..]` and `dst1 = &mut col_(j+1)[j+1..]`
  share identical SIMD body/tail boundaries (both length `nrow - j - 1`)
  and identical src memory locations per row. Bit-exact with rank-1
  per-element rounding because each SIMD lane op = scalar f64 op.
