### Parity — un-ignored 7 panel matrices, filed one real regression

Reran the 13 `#[ignore]`'d parity tests cold against current
`main`. Two passed under the MUMPS-only gate (CERI651DLS_0618 and
ROSZMAN1_0241) and were un-ignored directly.

Switched `tests/parity.rs` from MUMPS-only to oracle-consensus:
feral inertia must match **either** MUMPS 5.8.2 **or** SPRAL SSIDS.
This is verbatim the CLAUDE.md contract:

> Inertia must be exactly correct on non-singular matrices. On
> matrices where the canonical Fortran direct solvers (MUMPS 5.8.2
> and SPRAL SSIDS) disagree on inertia, feral must agree with at
> least one of them.

the new gate and a `try_read_oracle()` helper. Hand-edited
`tests/parity.rs` to match (the example was non-runnable —
`autoexamples = false` in `Cargo.toml`).

### v0.4.0 on PyPI — re-cut clean after the first publish broke

Tagged + released v0.4.0. The first wheel run (26002981755) failed
4/4 jobs. Three layers of bug surfaced in sequence — none had ever
fired before because the wheel matrix only runs on
`release`/`workflow_dispatch`:

1. `CIBW_BUILD_FRONTEND: "build[uv]"` requires `uv` on the runner;
   macOS-14 and windows-latest don't ship it. Dropped to plain
   `"build"`. (commit 07d385e)
2. `CIBW_BEFORE_BUILD_LINUX` invoked `rustup` but the manylinux
   container ships without it. Bootstrapped via the `sh.rustup.rs`
   installer. (same commit)
3. **The real failure on Linux + macOS:** cibuildwheel only copies
   `python/` into the sandbox, so `feral = { path = ".." }` in
   `python/Cargo.toml` resolves to `/Cargo.toml` and dies with
   `failed to load manifest for dependency feral`. Switched the
   wheel matrix to `PyO3/maturin-action@v1`, which mounts the whole
   `$GITHUB_WORKSPACE` into the manylinux container. (commit
   2442d1f)

Verification dispatch run 26003260115 went green on all 4 wheels
+ smoke-test.

Re-cut v0.4.0 cleanly: `gh release delete v0.4.0 --cleanup-tag`,
re-tagged at 2442d1f, re-published. Release run 26003542088

## Decisions Made

None new. The oracle-consensus parity gate is an *implementation*

## Abandoned Approaches

Two attempts at fixing the v0.4.0 wheel pipeline were abandoned in
favor of `PyO3/maturin-action`:

1. cibuildwheel with `build[uv]` frontend — required `uv` on every
   runner (macOS, Windows don't ship it). Dropped to plain `build`.
2. cibuildwheel with rustup bootstrap inside `CIBW_BEFORE_BUILD_LINUX`
   — fixed rustup but did not fix the `path = ".."` workspace
   problem; cibuildwheel ships only the package dir into the
   container.

The deeper lesson — recorded inline in the workflow comment — is
that cibuildwheel is the wrong tool when the Python crate has a
path-dependency on a sibling Rust crate. `maturin-action` mounts
`$GITHUB_WORKSPACE` and just works. See
`dev/tried-and-rejected.md` for the append-only entry.
