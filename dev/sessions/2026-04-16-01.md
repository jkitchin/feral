### Commit 3 — AmdWorkspace + init fast paths (`dd6eee9`)
New module `crates/feral-amd/src/workspace.rs` (~460 lines incl.
10 unit tests). Exposes:
- `const NONE: i32 = -1`, `flip(x) = -2 - x`, `clear_flag(wflg, wbig,
  w)` helper (matches faer `amd.rs:130-143`).
- `AmdWorkspace` owning all 9 primary i32 arrays (pe, len, nv, elen,
  degree, w, head, next, last) plus `iw` of length `iwlen = nzaat +
  nzaat/5 + n`, and scalars (wflg, wbig, lemax, mindeg, ncmpa, nel,
  ndense, pfree, iwlen).
- `AmdWorkspace::new(pattern, opts)` — single-pass construction:
  diagonal-skipped column counts → `len`, allocate `iw`, second pass
  fills `pe`, classifies each var into zero-degree pre-eliminated /
  dense-deferred / LIFO head-insert at `head[deg]`.
- Overflow guards: `n ≥ i32::MAX`, `iwlen > i32::MAX` → `AmdError::
  IndexOverflow`.

Docstring correction on `AmdOptions::dense_alpha`: α<0 sets the
threshold to `n − 2`, not infinity (faer `amd.rs:173-177`). Hubs of
degree `n − 1` are still deferred even with α<0.

Evidence: 17 feral-amd tests pass (10 new in workspace, 7 in
pattern). Clippy clean with one scoped `#![allow(dead_code)]` in

## Decisions Made
- Ordering work lives in **workspace-member crates**, one per
  backend, not as replacements in `src/ordering/`. Slice-based input
  so no crate depends on feral's `CscPattern` / `FeralError`. Recorded
  by the fact that `Cargo.toml` now has `[workspace]` and
  `crates/feral-amd/` is a sibling of the feral package. Not yet
  separately recorded in `dev/decisions.md` — adding as a follow-up.
- Clean-room invariant enforced by `scripts/check-amd-cleanroom.sh`
  in CI. The SuiteSparse `amd` crate lives **only** in the external
  oracle harness and never in feral's dependency graph. The script
  excludes `ref/` (vendored reference copies like `ref/faer-rs` are
  not workspace members).
- `AmdOptions::dense_alpha < 0` inherits faer's `n − 2` semantics,
  not an infinite threshold. Documented.
